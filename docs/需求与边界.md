# 需求与边界

## 一、要做什么（In Scope）

- 搭建一个**基于 agent 的财务 AI 助手平台**，部署在公司 Windows 主机，内网浏览器多人访问。
- agent **自主规划**：理解自然语言诉求 → 选择并调用合适的技能 → 读取网盘数据 → 产出结果。
- 以**技能（skills）**的形式扩展能力，每个技能职责单一、可独立维护。
- 首批两个技能：
  1. **按客户名找银行回单**：检索网盘中指定客户的银行回单并导出。
  2. **按销售人员拆分 Excel**：把汇总表按销售人员拆成单独文件。
- 所有产物写入 `outputs/`，对源数据只读。

## 二、不做什么（Out of Scope）

- **不在本阶段写业务实现代码**——当前只搭骨架与文档。
- **不修改源数据**：不回写、不覆盖、不删除网盘上的任何原始文件。
- **不做死流程 / RPA 录制**：核心是会规划的 agent，不是固定按键脚本。
- **不出网处理敏感数据**：默认内网闭环，不上传财务数据到外部服务。
- **不替代人工审核**：产出供财务同事复核，关键动作（付款、对外发送）仍由人决定。
- **不做权限 / 账号体系的深度建设**（首版依托内网与主机本地控制，后续再议）。

## 三、用户是谁

| 角色 | 是谁 | 怎么用 |
|------|------|--------|
| 开发者 / 维护者 | 李明昊 | 搭建平台、编写与维护技能、处理数据接入与安全 |
| 业务使用者 | 财务部同事 | 内网浏览器里用自然语言提需求、取结果；培训后可做小幅调整 |
| 需求 / 验收方 | 陆经理（莎莎姐）等 | 提出诉求、确认方向、验收成果 |

## 四、各角色职责

- **开发者（李明昊）**
  - 设计平台架构、部署到 Windows 主机、保障内网可访问。
  - 编写、测试、维护每个技能；把控数据只读与产物隔离的安全底线。
  - 给财务同事做培训。
- **业务使用者（财务同事）**
  - 用自然语言提出诉求，提供必要输入（如客户名、文件位置）。
  - 复核 agent 产出；培训后可对技能做小幅调整（如改输出字段、文件名规则）。
- **需求 / 验收方（陆经理等）**
  - 明确业务诉求与优先级，验收成果。

## 五、安全与边界红线（与 README 一致）

1. 源数据**只读**。
2. 产物只写 `outputs/`。
3. 敏感数据、密钥、本地配置**不进 git**。
4. 默认**内网闭环**，不外传财务数据。
