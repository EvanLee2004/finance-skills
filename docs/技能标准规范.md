# 财务部 Agent 技能标准规范

> 定稿 2026-06-18，李明昊。**以后每个 skill 都按本规范做。** 依据：Anthropic 官方 Agent Skills 最佳实践 + 官方 skill-creator 规范 + 本部门约定。

## 一、核心原则：人说人话，脏活归 agent

一个合格的 skill，**不该让人改文件名、摆文件夹**。人在浏览器里说一句"跑本周应收账款"，**agent 自己**去找文件、按内容认文件、跑脚本、回复核报告。人当机器的预处理器 = 反面教材。

## 二、标准文件夹（四件套，每个 skill 都长这样）

```
skills/<name>/
├── SKILL.md          ← 必需。YAML 头(name + description) + 流程指令正文
├── scripts/          ← 确定性/重复活的 python（agent 直接跑，不占上下文）
├── references/       ← 详细文档(如 SOP)，按需才加载
└── config/           ← 会变的表：列名别名、维护表等（用户维护，规则变改表不改码）
```

## 三、SKILL.md 写法（官方铁律）

1. **渐进式披露**：三层加载——①name+description 常驻；②命中才读正文(<500 行)；③scripts/references **按需**才碰，**脚本能直接执行、不进上下文**。
2. **description = 生死线**：Claude 靠它从上百个 skill 里挑，写不好就**根本不触发**。公式：`做什么 + 何时用 + 触发词/场景`，要"主动一点"（防漏触发）。≤1024 字符、无 XML 标签。
3. **正文**：祈使句、讲清"为什么"而非堆 MUST；导航式不穷举；重复的活抽成 `scripts/` 写一次。

## 四、智能交互模型（输入怎么进 = 混合模式）

1. **人**：自然语言触发（"跑本周应收账款"）。
2. **agent 找+认**：默认去**约定的网盘/文件夹区**扫，**按内容识别**每个输入的角色（如：多年份 sheet=源台账；带销售标注列=回填源；含"离职销售/接手"列=维护表）——**不靠文件名**；找不到或有歧义**才问人**。
   - 实现：bundle 一个 `scripts/inspect_inputs.py`，sniff 每个 xlsx 的 sheet/表头 → 分类。
3. **agent 跑**：调 `scripts/` 里的确定性脚本执行流程。
4. **agent 回**：复核报告(#N/A、告警、被删行清单等) + 真要人定的决策点 → 问人，不瞎来。
5. **产物**：写到 `output/`（带时间戳防覆盖）。

## 五、会变 vs 固定（划界）

- **固定逻辑**写进脚本（口径死的算式、流程步骤）。
- **会变**外置成 `config/` 配置表（列名别名、业务归属维护表）——规则变=改表。
- **拿不准/没见过的情况** → agent 停下问人。

## 六、安全红线（与平台一致）

源数据只读；产物只写 `output/`；敏感数据/密钥/真实数据不进 git；删行类操作要留底清单可追溯；对外发送前人工过目。

## 七、验收标准（达到才算好 skill）

用**真实历史数据**跑，产物能**逐项复现已知正确成品**才算过（如应收 skill：复现 `2026.6.4应收all.xlsx` 的行数/分布/规则）。复现不了不上线。官方建议：用要部署的所有模型都测一遍。
