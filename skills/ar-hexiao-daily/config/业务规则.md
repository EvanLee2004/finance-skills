# 业务规则参数（可改）

> 改完参数后重跑脚本即可，不必改 Python。  
> **2026-07-23 明妹口径**：日常主战场从「第 6 步智云筛核销」起；回填只动盈亏「明细」；流转待办=空+部分。

## 批次

- **筛日**：按**核销日期**筛（常用昨天；当天核销还在变，默认不做当天）
- **与「建回款」无固定隔天关系**：销售何时核销我们不管；核了就进该核销日，没核就进更后的日期
- **当前年**：默认 2026（跨年老单判定用）

## 数据源优先级（2026-07-23）

- **第 6 步智云导出 / 拉数** = 核销金额、类型、SO/SOD 的**真相源**
- **第 7 步流转表** = 线下备份 + 对到账写单号 +「是否更新应收款」状态；**不能**当核销金额来源
- **回填（原第 8/9 步）必须从第 6 步结果来**，不是从第 7 步「推」出来

## 写盈亏表（2026-07-23）

- **只改「明细」sheet**，其它 sheet 一个字节不许动
- 允许写她指定的**长期工作副本**（新文件输出仍保留）；原件对照用「其他 sheet 总数」
- 部分核销计提列：**留空不写 0**
- **回填硬闸（2026-07-23 优化）**：`validate_plan` → `build_review_sheet` 出《回填审核单》→ **她口头确认** → `apply_to_copy --confirmed` 才写。无 `--confirmed` 脚本拒绝写入。

## 流转「是否更新应收款」（三态 · 明妹录音）

待办集合（重扫）：

- **空** → 要更新 / 重扫
- **部分** → 要更新 / 重扫（与空同等重要）
- **红色字体**（人眼）→ 未更新；月初贴交付后优先补
- **是** → 已完成，默认不再动

**三态判据（2026-07-24 明妹口径修正）**：只看这笔到账下的订单**今天能不能在盈亏表里更新**：

| 这笔到账的订单情况 | 「是否更新」 |
|------|------|
| 全部订单都在她表里、这次都能填（含只是要**拆行/指认**的部分核销 E5/E8）| **是** |
| 有的能填、有的**还没交付进表**（E2/E3）| **部分**（未更新的标红）|
| 一个都还没交付进表 | **留空** |

> ⚠ 2026-07-24 关键修正（明妹 7-23 原话："部分是指某些订单没交付、我没法更新"）：
> **「部分核销」(E5) 不再让一笔到账掉成「部分」**。她那笔的钱已经全部核销落地，只是要在
> 表里把一个订单拆成「已收/未收」两行——这算她**更新了**。以前把 E5 也算成"没更新"，
> 导致一笔明明全核掉的到账被误标「部分」（实测 AR26070112：12 单里 1 单是 E5 部分核销 →
> 旧版「部分」、新版「是」）。**只有「订单没交付进表」(E2/E3) 才触发「部分」。**

写回建议（第一版**只出建议，不自动写流转表**）：

| 条件 | 「是否更新」填法 | 公式 | 颜色 |
|------|------------------|------|------|
| 关联订单**全部**没交付进表 / 都不能更 | **留空** | **不要公式** | — |
| 关联订单**全部**能更（含 E5 拆行）| **是** | 可套她表惯用公式，否则字面「是」 | — |
| 有的能更、有的**没交付进表** | **部分** | 不要写成「是」 | **没交付的 SO 标红** |

## 部分回款（没回满）怎么拆行（2026-07-24 明妹口径落码）

一笔到账没把订单收满时（E5），程序**把两行的具体金额算给她**（不再只说一句"插一行"）：

- **【已收行】**：回款明细填 = 本次实到额；**是否结账 = 是**；**计提留空**（没回满不填计提）
- **【未收行】**：在已收行上方**新增一行**，应收 = 差额（该收 − 实到）；**是否结账 = 否**；其余留空

> 实测 AR26070105：该收 8946.89 / 实到 8729.35 / 差 217.54 → 已收行 8729.35 结账「是」、
> 新增未收行 217.54 结账「否」。她当天就是这么手工做的。
> **注**：程序只把这套方案写进《核销日清》让她照做，**不自动往她表里插行**——插行会动到
> 她表里的透视/公式/图形对象引用，风险高；拆行仍由她手动执行（与"只改命中行几个格、其余
> 字节不动"的写入安全铁律一致）。

## 计提金额什么时候填（2026-07-23 明妹原话确认）

> 「一个订单的金额分成多次回款，**只有回款明细金额加起来等于交付金额之后**，才可以填写计提金额」

- **判据 = 该 SOD 的累计回款明细 == 智云的交付金额** → 填计提；否则**留空**
- **基准是智云交付额，不是她表里的应收金额**。实测 SO26040322 行2567：
  她表应收 488.64 / 回款明细 477.61 / 智云交付 477.61 → 她照样填了计提 477.61
- **「结账」和「计提」是两个判据**：结账看这一行的钱到没到（到了就填「是」），
  计提看整单回满没有。实测 SO26020068（回 8729.35 / 交付 8946.89）、
  SO26030424（回 199.78 / 交付 402.26）都是 **结账=是、计提留空**

## 交付额中途变了：一律以智云为准（2026-07-24 明妹口径）

明妹原话："要让他动脑子去检查交付额，别以我表里那个为准。判断上面交付额跟下面核销额一不一样，
一样就以它为准，按实际核销金额回填。"（口述 SO107/422：初始交付 → 结算时变了）

- **写进盈亏表的值 = 智云本次核销/交付额**（回款明细、计提都用它），**永不以她表里那行的旧应收为准**。
- **命中行后若智云额 ≠ 她表应收**：一律顶一个 ⚠ 到《核销日清》「怎么办」列，写"已按智云额填、填前扫一眼"。
  以前只有「整段等比例差(×0.97)」会提醒，靠 SOD / SO 唯一行命中的（她没改过应收那几行）**闷声就填、不吭声**
  —— 实测 SO26040322 就有 2 行这样静默填（智云 477.61/661.31 vs 她表 488.64/676.58）。现在每一行都提醒。
- **「超额核销」E4 的上限 = 智云该单交付额**，不是她表应收。她表应收可能是旧值；交付额变**大**时
  （初始 1 → 结算 2），拿旧应收当上限会把"本该填的 2"误报成超额。只有 `本次核销 > 智云交付额` 才是真超额。

## 核销状态

- **完整核销（可结账）**：手动核销、自动核销、核销成功、预存已核销
- **部分 / 未满**：预存部分核销、核销确认中、销售待核销、预存待核销、待提交
- **作废**：已作废 → 异常不写

## 取数：不再按回款类型分通道

**所有类型走同一条路**：回款记录（核销日期=T-1）→ 关联「下单」拿 SO+交付额 →
关联「订单同币种核销明细信息」拿逐 SO 核销额（**0 行 = 全额核销**，且累积表须按核销日期过滤）→
由 SO 查「订单明细」表拿 SOD + 逐 SOD 交付额。

> ⚠ 旧的「按回款类型路由三通道」已废止：预收类既被踢出对账通道、又在明细子表拿不到行，
> 两头落空还不报错 —— 2026-07-22 真跑**静默丢了 3 笔到账 6.5 万**。

| 回款类型 | 现在怎么处理 |
|---|---|
| 整笔 / 自动核销 / 预存 / 预收 / （空） | 同一条路，按 SOD 落行 |
| 分笔回款 | **一律 hold（E1）**，系统无逐单金额，禁止比例分摊 |

## 收款时间（R5）

- 核销日与到账日 **同月** → 填 **到账日期**
- **跨月** → 填 **核销日期**

## 匹配

- 先 SOD（对盈亏「实收金额」列），再 SO（对「新智云单号」）
- SO 命中多行 → 挂起（E8），列出候选
- 表中找不到 → hold E2；单号年份 < 当前年 → E3
- **禁止用行号定位**

## 红线（不可改）

- 金额只由脚本算
- 分笔永远 hold，禁止均分/按比例
- 永不写智云
- 回填只动盈亏「明细」sheet；其它 sheet 一个字节不动。可写工作副本（`--in-place` 就地写同一份，或默认出新文件）——两种都**写前自动备份 + 写后逐格回读校验**，校验不过绝不替换她的表
- **无人工确认（`--confirmed`）禁止写入**
