# 业务规则参数（可改）

> 改完参数后重跑脚本即可，不必改 Python。  
> **2026-07-23 明妹口径**：日常主战场从「第 6 步智云筛核销」起；回填只动盈亏「明细」；流转待办=空+部分。

## 批次

- **筛日**：按**核销日期**筛（常用昨天；当天核销还在变，默认不做当天）
- **与「建回款」无固定隔天关系**：销售何时核销我们不管；核了就进该核销日，没核就进更后的日期
- **当前年**：默认 2026（跨年老单判定用）

## 数据源优先级（2026-07-23）

- **第 6 步智云导出 / 拉数** = 核销金额、类型、SO/SOD 的**真相源**
- **第 7 步流转表** = 线下备份 + 对到账写单号 +「是否更新应收款」状态；**不能**当核销金额来源
- **回填（原第 8/9 步）必须从第 6 步结果来**，不是从第 7 步「推」出来

## 写盈亏表（2026-07-23）

- **只改「明细」sheet**，其它 sheet 一个字节不许动
- 允许写她指定的**长期工作副本**（新文件输出仍保留）；原件对照用「其他 sheet 总数」
- 部分核销计提列：**留空不写 0**

## 流转「是否更新应收款」待办集合

- **空** → 要更新 / 重扫
- **部分** → 要更新 / 重扫（与空同等重要）
- **红色字体**（人眼）→ 未更新；月初贴交付后优先补
- **是** → 已完成，默认不再动

## 尾差

- **尾差阈值（元）**：`0`
  - 默认不容忍任何差额；到账与交付/核销对不上 → 一律挂起（E5）
  - 待明妹确认后再改（实测见过 0.12、0.32）

## 手续费

- **手续费口径**：未拍板
  - 有手续费 > 0 的笔 → **一律挂起**，不自动回填金额
  - 待确认：用到账净额还是含手续费额

## 收款方式取值

- 预存类核销状态（预存已核销 / 预存部分核销 / 预存待核销）→ `冲预收`
- 其余 → `汇`
- 表中只用这两个值

## 核销状态

- **完整核销（可结账）**：手动核销、自动核销、核销成功、预存已核销
- **部分 / 未满**：预存部分核销、核销确认中、销售待核销、预存待核销、待提交
- **作废**：已作废 → 异常不写

## 回款类型 → 取数通道

| 回款类型 | 通道 |
|---|---|
| 整笔回款 / 自动核销 / （空） | 回款核销对账表 |
| 预存回款 / 预收回款 | 核销明细子表（**必须按核销日期过滤本次**） |
| 分笔回款 | **无数据 → 一律 hold（E1）**，禁止比例分摊 |

## 收款时间（R5）

- 核销日与到账日 **同月** → 填 **到账日期**
- **跨月** → 填 **核销日期**

## 匹配

- 先 SOD（对盈亏「实收金额」列），再 SO（对「新智云单号」）
- SO 命中多行 → 挂起（E8），列出候选
- 表中找不到 → hold E2；单号年份 < 当前年 → E3
- **禁止用行号定位**

## 红线（不可改）

- 金额只由脚本算
- 分笔永远 hold，禁止均分/按比例
- 永不写智云
- 回填只动盈亏「明细」sheet；其它 sheet 一个字节不动。可写工作副本（`--in-place` 就地写同一份，或默认出新文件）——两种都**写前自动备份 + 写后逐格回读校验**，校验不过绝不替换她的表
