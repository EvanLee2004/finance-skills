## 一、取数：只有一个入口

财务管理 →「**回款记录**」，**只筛一个条件：核销日期 = T-1**。
然后把每笔回款记录**点进去的明细页**抓下来——这就是明妹手工在做的动作。

```
回款记录（1 笔到账）
  ├─ 关联「下单」                    → 这笔关联了哪几个 SO + 每个 SO 的交付额
  ├─ 关联「订单同币种核销明细信息」  → 逐 SO 的本次核销金额
  │                                    ⚠ 0 行 = 全额核销（明妹原话：没有这张就说明到账=交付）
  │                                    ⚠ 该子表是**累积**的，必须按本次核销日期过滤
  │                                    ⚠ 表里没有叫 SO 的字段，SO 在关联字段「订单NUM」里
  │                                    ⚠ 「是否已撤销=是」的行不算数
  └─ 由 SO 去「订单明细」表          → 每个 SO 下的 SOD + 逐 SOD 交付额
                                       **SOD 的唯一来源**；她盈亏表一行 = 一个 SOD
```

### 已废弃：按核销日期单独筛「回款核销对账」表

三条理由（2026-07-23 实调）：

1. **它对预存/预收类返回 0 行**。旧版靠"按回款类型走三条通道"来补，而通道路由正是
   2026-07-22 **静默丢 3 笔到账 6.5 万**的根因（预收类既被踢出对账通道、又在明细子表
   拿不到行，两头落空还不报错）。
2. **它拿不到 SOD 与逐 SOD 交付额**，判不出"一个 SO 拆 N 个 SOD"这个主场景。
3. 它是独立筛的第二张表，与回款记录可能不同步；关联读法天然同步。

> 顺带销掉一条旧卡点：「同币种核销明细」用陆总账号 `getFilterRows` **直查 0 行**，
> 但用 `getRowRelationRows` **关联读得到**。这不是账号权限问题，是查法问题。

## 二、判定

```
每笔到账：
  A = 到账金额（本币；系统给了就用系统的）
  H[so] = 同币种核销明细逐 SO 核销额；该表本次 0 行 → H[so] = 该 SO 交付额

  ── 校验 ΣH ≤ A + 手续费        她的铁律：销的钱不能比到的钱多
       超了且有核销明细 → E4 超额核销（智云脏数据，找销售）
       超了且无核销明细 → E5 这单没回满（提示她插行拆「已收/未收」）
  ── 校验 每笔到账至少产出 1 条判定，否则**整体报错退出**（防静默丢单）

  每个 SO：
    lines = 订单明细[SO]（SOD + 交付额）
    H[so] == Σlines  → 该 SO 全部 SOD 入选，每个填自己的交付额
    否则             → 在 lines 里凑**唯一**子集；多解/无解/行太多 → E5 挂起并列候选
    lines 为空       → 退化按 SO 匹配盈亏表（仍可判，不是丢单）
```

## 三、盈亏表定位（她表里一行 = 一个 SOD）

优先级：

1. **SO + 应收金额** ← 主键。回填前后都成立（应收金额 = 该 SOD 交付额）
2. **SOD** ← 她已回填过「实收金额」列时可用（幂等重跑走这条）
3. **SO 唯一行**
4. 同一 SO 下多行金额相同 → 试**整段逐位对齐**：
   她这个 SO 的全部行（行号升序） vs 智云该 SO 的全部 SOD（编号降序），
   金额序列**逐位完全相等**才算对齐（两边都源自月初同一份「交付数据」，天然同序）。
   有一位对不上就老实挂 E8。
5. 都不行 → 表里找不到这单：SO 是往年的 → E3 跨年老单；否则 E2 还没交付。
   **注意顺序：先查表、查不到才判 E3**。2025 年的单在 2026 表里有行是常态。

## 四、五列怎么填

| 列 | 值 |
|---|---|
| 计提金额 | 该 SOD 交付额（核销状态属"已完整核销"时；否则留空） |
| 回款明细 | 该 SOD 交付额 |
| 是否结账 | 是（完整核销）/ 否 |
| 收款时间 | **到账日期**（同月）；跨月填核销日期 |
| 收款方式 | **汇**；只有「到账月 ≠ 核销月」才填**冲预收** |
| 实收金额 | 该 SOD 号 |

> ⚠ 2026-07-23 删掉旧规则「预存/预收类 → 冲预收」。实测证伪：7-22 那天
> AR26070089/107/112 全是预存回款，她 15 行**全填「汇」**；她整张 2026 表 2921 行里
> 「冲预收」只有 175 行（6%），不可能等于预存类占比。旧规则会把这 15 行全填错。

## 五、仍然一律 hold 的

- **分笔回款**：系统没有逐单金额，铁律挂起，**禁止**均分 / 按交付比例 / 估算。
- **有手续费**：净额/含费口径未拍板。
