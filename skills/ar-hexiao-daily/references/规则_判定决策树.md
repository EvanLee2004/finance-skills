# 规则 · 判定决策树（v2 · 2026-07-23）

> 取数与口径全文见 `规则_取数与判定.md`；E 码文案见 `config/判定码.json`。
> **不再有"按回款类型分三通道"这一步**——那是 2026-07-22 静默丢 3 笔到账的根因。

## 一、按「每笔到账」先定性

```
分笔回款 ────────────────────→ hold E1（系统无逐单金额，禁止比例分摊）
手续费 > 0 ──────────────────→ hold E_FEE（净额/含费口径待拍板）
外币且无汇率 ────────────────→ hold E6
下单栏为空 ──────────────────→ exception E7（这笔回款在智云没挂订单）
ΣH > 到账 + 手续费
   ├ 有核销明细 ─────────────→ exception E4（超额核销，找销售）
   └ 无核销明细 ─────────────→ hold E5（这单没回满，提示插行拆「已收/未收」）
否则 ────────────────────────→ 进第二步逐 SO 展开
```

**硬校验**：每笔到账至少产出 1 条判定；`expand_payments` 出口比对
`{到账的 AR}` 与 `{判定的 AR}`，不等就 `CoverageError` 报错退出。

## 二、逐 SO → SOD

```
H[so] = 核销明细的本次核销额（该表本次 0 行 → 该 SO 交付额）
lines = 订单明细[SO]

H[so] == Σlines ──────────→ 全部 SOD 入选
凑得出唯一子集 ───────────→ 只填这几个 SOD
多解 / 无解 / SOD > 22 行 ─→ hold E5，把候选 SOD 与金额摆出来给她挑
lines 为空 ───────────────→ 退化：按 SO 匹配盈亏表（仍可判）
```

## 三、每个 SOD → 盈亏表定位

```
① SO + 应收金额  → 唯一行 ✅（主键：回填前后都成立）
② SOD            → 唯一行 ✅（她回填过「实收金额」列时；幂等重跑走这条）
③ SO             → 唯一行 ✅
④ 多行同额       → 整段逐位对齐（SOD 降序 ↔ 行号升序，金额序列全等）✅
                   对不齐 → hold E8，候选行号仅供参考
⑤ 表里没有       → SO 是往年 → hold E3；否则 hold E2
⑥ 本次两条命中同一行 → 两条都 hold E8（谁对谁错要人定）
本行应收 < 本次金额 → exception E4
流转三键 0 命中 / 多命中 → E0 / E12（仅在 --flow-complete 时判 E0）
```

## 四、五列（R4 / R5 / R6）

| 列 | 完整核销 | 部分核销 |
|---|---|---|
| 计提金额 | 该 SOD 交付额 | 留空 |
| 回款明细 | 该 SOD 交付额 | 本次金额 |
| 是否结账 | 是 | 否 |
| 收款时间 | **R5**：同月填到账日 / 跨月填核销日 | 同 |
| 收款方式 | **R6**：默认「汇」；**只有到账月≠核销月**才「冲预收」 | 同 |
| 实收金额列 | 写 **SOD** | 同 |

> R6 已于 2026-07-23 删掉「预存/预收类→冲预收」那条（15 笔实测全错）。详见 `规则_取数与判定.md` §四。
