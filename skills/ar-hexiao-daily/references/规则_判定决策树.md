# 规则 · 判定决策树（R1–R9 + E0–E12）

> 完整业务依据见项目方案文档 `20260722_核销回填规则与例外目录.md`。  
> ⚠ 2026-07-22 实测：对账表只覆盖约 61%；预存走明细子表；分笔两边皆无数据。

## 入口

1. 拉 T-1 核销（以**回款记录**为入口，按 `回款类型` 分通道）  
2. 可选：流转表三键（日期+金额+公司名）对到账 → 对不上 E0/E12  
3. SO 是否在当年盈亏明细 → 否 E2；跨年 E3  
4. 类型分支：
   - **分笔** → hold E1（禁止比例分摊）
   - **预存** → 明细子表，**按核销日期过滤本次**
   - **整笔/自动** → 对账表逐 SO 金额
5. 手续费 > 0 → hold E_FEE（口径待拍板）  
6. 尾差超阈值（默认 0）→ hold E5  
7. 外币无汇率 → E6  
8. 完整核销 → auto 五列；部分 → 清单建议（第一版不加行）

## 五列（R4）

| 列 | 完整核销 | 部分 |
|---|---|---|
| 计提 | = 到账本币 | 不填 |
| 回款明细 | 实收本币 | 本次/累计 |
| 是否结账 | 是 | 否 |
| 收款时间 | R5 | R5 |
| 收款方式 | 汇/冲预收 | 同 |
| 实收金额列 | **写 SOD** | 同 |

## 收款时间 R5

- 同月 → 到账日  
- 跨月 → 核销日
