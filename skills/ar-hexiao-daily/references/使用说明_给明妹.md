# 应收核销日清 · 使用说明（给明妹）

> 读者是出纳，不是程序员。照着做就行。

---

## 1. 一次性准备

1. 电脑上已有本技能文件夹（同事会帮你放好）。  
2. 需要 Python 和 `openpyxl`（同事装好；你一般不用管）。  
3. 记住工作区长这样：

```
工作区/
  01_智云导出/     ← 你从智云导出的表放这里
  02_我的表副本/   ← 盈亏表、流转表「另存为」副本放这里（原件别直接丢进来）
  03_台账/         ← 电脑自动维护的挂账台账
  04_产出/         ← 每天的清单和报告从这里拿
```

4. 第一次可先让同事帮你跑一遍「盘点」，看文件夹是否认得出。

---

## 2. 每天怎么做（主战场从「筛核销」开始）

> 前面日记账 / 挑收入 / 建回款你照旧自己做就行；**自动化重点从智云筛核销开始**。  
> 销售何时核销不用管——筛的是**核销日期**，核了就进那天，没核就进更后的日子。

1. **智云数据（真相源）**  
   - **手导**：按「核销日期 = 昨天」导出回款记录 / 对账 /（预存）明细 → `01_智云导出/`  
   - **或**说「拉智云」，当场输账密（只读、不存密码）

2. **盈亏工作副本**  
   固定用一份**副本**（原件自己留着对照）。程序**只改「明细」这一个 sheet**，其它 sheet 不动。  
   你怎么验：看副本**其它 sheet 底下总数**是否还和原版一致。

3. **流转表副本**  
   放进 `02_我的表副本/`，用来对到账、写单号。  
   **「是否更新应收款」：空 和 「部分」都要再做**；红字=没更新，月初贴完交付优先补。

4. **说「跑昨天的核销」**  
   判定（以智云为准）→ 工作清单 → **回填审核单**（到这里你的表还没被改）。

5. **你打开《回填审核单》扫一眼**  
   看「盈亏回填·拟写入」：准备改哪几列、改成啥、依据是啥。  
   没问题回一句 **「确认」或「可以写」** → 电脑才写入盈亏副本「明细」（写前自动备份）。  
   有问题说「跳过某某 SO」或「这笔不对」。

6. **挂账**  
   挂着的说「重扫挂账」。

## 3. 清单怎么看

### A. 工作清单（总览）

#### 页签「今天能填」

电脑认为可以回填的笔。每一列：

| 列 | 意思 |
|---|---|
| AR / SO / SOD | 回款号、订单号、结算号 |
| 客户(打码) | 只显示打码，保护隐私 |
| 怎么找到这行 | **按 SO 号在盈亏表筛选**，不要按行号（表会插行，行号会变） |
| 应填_计提 / 回款明细 / 是否结账 / 收款时间 / 收款方式 | 建议填什么 |
| 当前_* | 你表里现在是什么；对不上说明表变了，别盲填 |
| 判定依据 | 一句话为什么这样判 |

#### 页签「挂账待办」

暂时不能填盈亏的（比如分笔、还没交付、跨年老单）。先可在流转表写单号，盈亏等条件具备再补。

#### 页签「异常」

数据有问题（对不上账、超额、缺汇率等），需要你或销售先查清楚。

#### 页签「按到账汇总」

到账流转表「是否更新应收款」建议：

| 建议 | 你怎么填 |
|------|----------|
| （空白）/ 空 | **留空**，不要写公式 |
| 是 | 全部订单已更新 → 填「是」（有惯用公式可套公式） |
| 部分 | 有的更了有的没 → 填「部分」，**没更新的单号标红** |

### B. 回填审核单（写表前必看 · 核心）

| 页签 | 干什么 |
|------|--------|
| 使用说明 | 三步确认法 |
| **盈亏回填·拟写入** | **改前→改后**、判定依据、风险；确认后才写这些 |
| 幂等跳过 | 已经一样，再跑也不会改 |
| 冲突·需你定 | 对不上，程序不会自动写 |
| 流转表·是否更新建议 | 只建议，**不自动改你的流转表** |

## 4. 每个异常原因怎么办

（大白话，不记代号）

| 你看到的话 | 你该做什么 | 找谁 |
|---|---|---|
| 分笔回款，系统没有拆到每张单的金额 | 先挂着；流转表可只写单号；等回满或销售补清再补盈亏 | 销售 |
| 盈亏表里还没有这张单 | 等多交付贴进明细，再说重扫挂账 | 自己 / 月初流程 |
| 往年的单，今年表里没有 | 按老单人工流程，或查往年表 | 自己 / 主管 |
| 核销金额比到账还多 | 先别填，查系统是不是录错 | 销售 |
| 金额差了一点点也对不上 | 先挂着，人工看是不是尾差（规则还没定宽松阈值） | 自己 |
| 外币缺汇率 | 补当天中行牌价后再跑 | 自己 / 同事 |
| 没有单号或已作废 | 跳过；无单号问销售 | 销售 |
| 这张单在表里有好几行 | 你指定记哪一行再填 | 自己 |
| 带手续费，口径还没定 | 先挂着，等口径确认 | 明妹拍板 / 主管 |
| 到账流转对不上 | 查日记账有没有漏登 | 自己 |

## 5. 哪些它不会做

- **不替你录智云**（永不自动往系统写数据）。  
- **不碰你盈亏表的原版、也不碰「明细」以外的任何 sheet**（原版你留着比总数验收）。回填只往你固定用的那份**工作副本的「明细」**里写，写前先自动备份一份。  
- **你没说「确认」之前，绝不写你的表**（先出审核单给你看）。  
- **不自动改到账流转表**（单号 / 是否更新只给建议，你手填或以后再加）。  
- **分笔回款**判不了金额（大约一成），只能挂起。  
- **部分核销要加行**的，暂不自动加行，只提示。  
- 不替销售做核销。

---

## 6. 出错了怎么办

1. 看弹窗或对话里的「ERROR」全文。  
2. 若提示「表头缺列…实际表头：…」——说明导出的列名和预设对不上，把报错截图给同事，别自己猜列。  
3. 日志/报告在 `04_产出/运行报告_*.txt`。  
4. 找谁：先找部署技能的同事（明昊侧）；业务口径找你自己或主管。

## 7. 常见问题

**Q：导出的表头和以前不一样？**  
A：程序按中文列名模糊认。认不出就报错停下，不会瞎猜。把实际表头发给同事加别名即可。

**Q：跑一半停了？**  
A：看报错。改完文件后从断的那步重跑；判定和清单可重复跑。

**Q：同一天跑两遍会不会重复？**  
A：挂账重扫同一天再跑不会叠次数；清单会覆盖生成。你自己往表里填时注意别重复累加——清单上有「当前值」可对照。

**Q：预存类准不准？**  
A：本地测试包几乎没有预存样例，程序已按规则写了（含按核销日期过滤），上线后要用你真实导出再验。同事不会假装「已经验过」。

**Q：收入提取那步？**  
A：把银行日记账给程序，它会挑出纯销售收入清单；空预填行会自动跳过；美元和人民币不会混加。
