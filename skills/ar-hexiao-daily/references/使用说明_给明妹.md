# 应收核销日清 · 使用说明（给明妹）

> 读者是出纳，不是程序员。照着做就行。
> **2026-07-23 改版**：产出从两个文件九个页签，合并成**一份《核销日清》**；
> 第一列是单号，第二列是状态，问题类型直接变成一列。

---

## 1. 一次性准备

1. 电脑上已有本技能文件夹（同事会帮你放好）。
2. 需要 Python 和 `openpyxl`（同事装好；你一般不用管）。
3. 记住工作区长这样：

```
工作区/
  01_智云导出/     ← 从智云取的表放这里（程序自己拉，也可以你手导）
  02_我的表副本/   ← 盈亏表、流转表「另存为」副本放这里（原件别直接丢进来）
  03_台账/         ← 电脑自动维护的挂账台账
  04_产出/         ← 每天的《核销日清》从这里拿
```

4. 第一次可先让同事帮你跑一遍「盘点」，看文件夹是否认得出。

---

## 2. 每天怎么做（主战场从「筛核销」开始）

> 前面日记账 / 挑收入 / 建回款你照旧自己做；**自动化重点从智云筛核销开始**。
> 销售何时核销不用管——筛的是**核销日期**，核了就进那天，没核就进更后的日子。

1. **智云数据（真相源）**
   - 说「**拉智云**」，当场输账密（只读、不存密码）。它自己去财务管理→回款记录，
     **只按「核销日期 = 昨天」筛这一个条件**，再把每笔点进去的明细页也抓下来。
   - 或者你手导也行，把导出的表放进 `01_智云导出/`。

2. **盈亏工作副本**
   固定用一份**副本**（原件自己留着对照）。程序**只改「明细」这一个 sheet**，其它 sheet 不动。
   你怎么验：看副本**其它 sheet 底下总数**是否还和原版一致。

3. **流转表副本**
   放进 `02_我的表副本/`，用来对到账、写单号。
   **「是否更新应收款」：空 和 「部分」都要再做**；红字=没更新，月初贴完交付优先补。

4. **说「跑昨天的核销」**
   它会取数 → 判定 → 校验 → 出**一份《核销日清》**。到这里你的表还一个字节没被改。

5. **打开《核销日清》扫一眼**
   按「状态」列筛「今天要填」，照「应填_xxx」几列填；
   没问题回一句 **「确认」或「可以写」** → 电脑才写进盈亏副本「明细」（写前自动备份）。
   有问题就说「跳过某某 SO」或「这笔不对」。

6. **挂账**
   挂着的处理完了，说「重扫挂账」。

## 3. 《核销日清》怎么看

**只有这一份，别的表都不用开。**

### 页签「先看这里」

一屏看完：今天要填几行、跳过几行、冲突几行、挂账几行、异常几行，和怎么用。

### 页签「今日清单」（主表）

**一行 = 一个单号（SOD）。第一列单号，第二列状态。**

| 状态 | 意思 | 你要干嘛 |
|---|---|---|
| **今天要填** | 电脑算好了，你表里那行还是空的 | 照「应填_xxx」填进去 |
| **已填过·跳过** | 你之前填过，而且跟这次算的一样 | **不用管**（这就是你问的"我明明填过了"那类） |
| **冲突·需你定** | 你填过，但跟这次算的不一样 | 看「差异(一眼扫)」那列，你定按哪个来 |
| **挂账待办** | 今天填不了 | 看「怎么办」那列 |
| **异常** | 数据不对劲 | 先别填，按「怎么办」查清楚 |

其它列：

| 列 | 意思 |
|---|---|
| 怎么办 | 这一行该干嘛，一句话 |
| SO / 到账号(AR) / 客户(打码) | 定位信息，客户名只显示打码 |
| 怎么找到这行 | **按 SO 号在盈亏表筛选**，不要按行号（表会插行，行号会变） |
| 应填_计提 / 回款明细 / 是否结账 / 收款时间 / 收款方式 / 实收SOD | 建议填什么 |
| 当前_* | 你表里现在是什么，可对照 |
| 差异(一眼扫) | 只有「冲突」行才有：哪一列、表里是啥、这次算的是啥 |
| 判定依据 / 码 | 一句话为什么这样判 |

### 页签「流转表怎么填」

到账流转表「是否更新应收款」建议：

| 建议 | 你怎么填 |
|------|----------|
| （空白） | **留空**，不要写公式 |
| 是 | 全部订单已更新 → 填「是」（有惯用公式可套公式） |
| 部分 | 有的更了有的没 → 填「部分」，**没更新的单号标红** |

## 4. 挂账 / 异常怎么办

（大白话，不记代号）

| 你看到的话 | 你该做什么 | 找谁 |
|---|---|---|
| 分笔回款，系统没有拆到每张单的金额 | 先挂着；流转表可只写单号；等回满或销售补清再补盈亏 | 销售 |
| 盈亏表里还没有这张单 | 等月初贴完交付，再说「重扫挂账」 | 自己 / 月初流程 |
| 往年的单，今年表里没有 | 按老单人工流程，或查往年表 | 自己 / 主管 |
| 核销金额比到账还多 | 先别填，查系统是不是录错 | 销售 |
| **这单没回满**（交付 XXX、实际只到账 YYY） | 在这单上方插一行，把应收拆成「已收 / 未收」两行，未收那行结账填「否」；拆完说「重扫挂账」 | 自己 |
| **分不清这次核的是哪几个明细单** | 清单上列了候选 SOD 和金额，你指一下 | 自己 |
| 外币缺汇率 | 补当天中行牌价后再跑 | 自己 / 同事 |
| 没有单号或已作废 | 跳过；无单号问销售 | 销售 |
| 这张单在表里有好几行一样的金额 | 你指定记哪一行再填 | 自己 |
| 带手续费，口径还没定 | 先挂着，等口径确认 | 明妹拍板 / 主管 |
| 这笔到账在系统里没挂订单 | 去智云看这笔回款建对了没 | 自己 / 销售 |

## 5. 哪些它不会做

- **不替你录智云**（永不自动往系统写数据）。
- **不碰你盈亏表的原版、也不碰「明细」以外的任何 sheet**（原版你留着比总数验收）。
  回填只往你固定用的那份**工作副本的「明细」**里写，写前先自动备份一份。
- **你没说「确认」之前，绝不写你的表**（先出清单给你看）。
- **不自动改到账流转表**（单号 / 是否更新只给建议，你手填或以后再加）。
- **分笔回款**判不了金额，只能挂起。
- **要拆行的**（钱没到齐那种）只提示，不替你插行。
- 不替销售做核销。

---

## 6. 出错了怎么办

1. 看弹窗或对话里的「ERROR」全文。
2. 若提示「表头缺列…实际表头：…」——说明导出的列名和预设对不上，把报错截图给同事，别自己猜列。
3. 日志/报告在 `04_产出/运行报告_*.txt`。
4. 找谁：先找部署技能的同事（明昊侧）；业务口径找你自己或主管。

## 7. 常见问题

**Q：导出的表头和以前不一样？**
A：程序按中文列名模糊认。认不出就报错停下，不会瞎猜。把实际表头发给同事加别名即可。

**Q：跑一半停了？**
A：看报错。改完文件后从断的那步重跑；判定和清单可重复跑。

**Q：同一天跑两遍会不会重复？**
A：不会。第二遍会认出「已填过·跳过」，不会再让你填一遍，也不会重复写。

**Q：怎么知道它算得对不对？**
A：拿你 7 月 22 号那天的真实数据完整回放过一遍——13 笔到账拆成 157 行，
其中 **143 行程序算的和你当天手工填的逐格一模一样**，一处都没差；
剩下 14 行是真填不了的（3 张单还没交付、8 张 2025 年老单、2 处要你先拆行、1 张 7 月新单没交付）。

**Q：预存类准不准？**
A：7-22 那天三笔预存回款（含一笔部分核销）都在上面那次回放里，算对了。
但**样本只有一天**，头几天还是并排看一眼再确认。

**Q：收入提取那步？**
A：把银行日记账给程序，它会挑出纯销售收入清单；空预填行会自动跳过；美元和人民币不会混加。
