# 应收核销日清（只判不写 · 第一版）

**出纳每天一条龙**：人在环中。第一版脚本接管 **第 1(可选) / 2 / 6 / 8 步**；第 7 步只**算清单不写她的表**；其余步骤她做，技能讲清怎么做。

**产出**：`工作区/04_产出/今日工作清单_*.xlsx` + 运行报告；挂账台账在 `工作区/03_台账/`。  
**数据只在本机。永不写智云。第一版不改她的任何 Excel。**

## 0. 先做这件事：按她的话直接跳到对应步骤（**不要从第 1 步顺着走**）

| 她说的话 | 直接做第几步 | 第一条命令 |
|---|---|---|
| 拉智云 / 抓昨天核销 / 导智云 / 自动导出 | **第 1 步** | `fetch_zhiyun.py --date yesterday --workspace 工作区/`（**运行时要账密**） |
| 跑昨天的核销 / 做核销 / 核销判定 / 出清单 | **第 6 步 → 第 7 步** | `classify_hexiao.py` 然后 `build_worklist.py` |
| 挑收入 / 提取收入 / 日记账 | **第 2 步** | `extract_income.py` |
| 挂账还有哪些没做 / 重扫挂账 / 月初补做 | **第 8 步** | `rescan_holds.py --ledger <盈亏副本>` |
| 我文件放好了 / 先看看缺什么 | **盘点** | `inspect_inputs.py` |

**「跑昨天的核销」= 第 6 步，不是第 2 步。** 挑收入是另一件事，她不提就别做。  
**第 1 步账密规则**：只从环境变量 `ZHIYUN_USER`/`ZHIYUN_PASS` 或交互输入拿；**禁止**写进 SKILL/config/对话回显密码；用完即丢。也可继续手导 Excel 放进 `01_智云导出/`，跳过第 1 步。

**判定+清单跑完，结尾必须照抄这段说给她（一个字都别改，尤其别说"填进智云"）：**

```
✅ 昨天的核销判定完了
📁 清单在：<今日工作清单_YYYYMMDD.xlsx 的绝对路径>
   · 「今天能填」N 笔 —— 打开清单，按里面给的 SO 号到**你的盈亏核算表**里筛选，照着「应填_*」几列填。
     （先对一眼「当前_*」列：对不上说明表变过了，别盲填。）
   · 「挂账待办」M 笔 —— 这次填不了，原因和"等什么"清单里都写了，已进挂账台账，以后我每天帮你重扫。
   · 「异常」K 笔 —— 需要找人的，清单里写了找谁。
   · 「按到账汇总」—— 每笔到账在流转表「是否更新应收款」该填 是 / 部分 / 空白。
🔒 你的表一个字节都没动（源文件校验已通过）
👉 下一步该你做：照清单填**盈亏核算表**和**到账流转表**；填完跟我说一声「填完了」
```

⛔ **绝不能说**"填进智云 / 录入智云 / 同步到系统"——**我们永远不碰智云**，她填的是**她自己的两张 Excel**。

## 0.5 硬禁止（实测踩过，违反即整单作废）

1. **禁止你自己写 Python/脚本去读她的 Excel**。只准调 `scripts/` 下已有的脚本。
   （实测：免费模型自己写了个 dump 脚本，把真实客户名和金额打印进了对话。）
2. **禁止把表格内容、客户名、金额明细回显到对话**——包括"我看一眼数据"式的探查输出。
3. **缺输入就停下报缺**，禁止改用别的文件凑合、禁止推测她该干嘛。
   照抄这句：`❌ 第 6 步跑不了：01_智云导出/ 里没有智云导出的 Excel（需要文件名含"回款"和"对账"的两份）。请导出后再说一声。`

## 0.8 判不准就**停下来问她**，不要默默挂起（组合 `task-clarifier`）

技能不是只会跑脚本的壳——**遇到下面这些，先问她一句，别自己猜也别一声不吭地挂起**。
问法照 `task-clarifier` 技能：**一次问 1–3 个、每个带选项、能一句话答**。

| 遇到什么 | 问她（带选项） |
|---|---|
| 脚本报「表头缺列」，列出了实际表头 | 「你导出的表里，`<实际列名>` 是不是我们要的 `<期望列名>`？ A 是 B 不是，是另一列 C 我截图给你看」 |
| 流转表匹配是**弱命中**（日期金额对上、公司名对不上） | 「7-14 这笔 47330.09，流转表上写的是「上海智信普惠」，智云开票客户是「XX」，**是同一笔吗**？A 是 B 不是 C 我看看再说」 |
| 同一个 SO 在盈亏表命中多行 | 「SO26010523 在你表里有 4 行，这次的钱记哪一行？ A 最早那行 B 还没结账那行 C 我自己来定」 |
| 有微信/支付宝手续费的笔 | 「这笔到账净额 298.38、手续费 1.62，智云核销记 300.00 —— **回填金额填哪个**？ A 298.38 B 300.00」 |
| 出现小额尾差（如 0.12 / 0.32） | 「差了 0.12 分，这种 A 忽略不管 B 也要挂应收 C 看情况我来定」 |
| 跨年老单（2024/2025 单号，今年表里没有） | 「这批老单 A 你自己手工处理，程序跳过 B 你把往年盈亏表给我，程序一起查」 |

**她答完就照答案跑，并把答案记进 `config/业务规则.md`**（下次别再问同一个问题）。
**她说"你先跳过"就挂起**——挂起也要在清单里写清楚"因为问了没定所以挂着"。

⛔ 反例（**不许**）：口径没定就自己挑一个填；表头对不上就找个相近的列凑；弱命中当强命中直接写进"今天能填"。

## 1. 红线（违反整单作废）

1. **金额只由** `scripts/*.py` **计算**，不许心算、不许对话凑数。  
2. **禁止把金额明细、客户名打进对话**；只报成功/失败/几笔/哪个 E 码。  
3. **分笔回款一律 hold**，禁止按比例分摊 / 均分 / 估算。  
4. **永不写智云**（无任何写接口调用；`fetch_zhiyun.py` 只读）。  
4b. **智云账密禁止写入仓库/配置/对话回显**；只允许环境变量或 getpass 运行时输入。
5. **第一版不写用户任何 Excel**：只读入，只写 `工作区/04_产出/` 与 `03_台账/`。  
6. **列名按中文模糊匹配**；找不到列 → 报错退出并列出实际表头。  
7. 判不了的一律 hold/异常，不许「尽量填一个」。

### 两个必须理解的开关

- **`--flow-complete`**：只有当她把**当天所有渠道**的到账流转表（汇款/微信/支付宝/美元户…）都放进 `02_我的表副本/` 时才加。
  不加 → 找不到对应到账时**不判 E0**，只标注"未在现有流转表中找到"。
  原因：她各渠道分表，只给一两张就判 E0 会把整批冤枉成"对不到账"（实测 53 笔会误报 28 笔）。
- **`--ledger`（rescan）**：给了才会**拿当前盈亏表重判挂起项**。月初她把交付数据贴进盈亏表后，
  上月因"表里还没这个 SO"挂起的笔就是靠这个自动变「可补做」——不给就只是合并，第 8 步等于白做。

规则细节 → `references/规则_*.md`；参数 → `config/`；给她看的操作 → `references/使用说明_给明妹.md`。

## 4. 你（agent）怎么干

1. 确认她把文件放进工作区（或先 `inspect_inputs`）。  
2. 按她说的步骤跑对应脚本；**金额结果只从脚本 stdout/产出文件读**。  
3. 对话只报：笔数、auto/hold/异常计数、E 码、路径、成功失败。  
4. 轮到她做的步：只讲做什么 + 怎么验，不代做。  
5. 装依赖：`pip install openpyxl xlrd`（**xlrd 必装**——她的银行日记账是老式 `.xls`，缺了第 2 步会直接崩）；第 1 步智云取数另需 `pip install requests playwright && playwright install chromium`。测试：`python3 -m pytest tests/ -v`。

## 5. 验收

`cd` 到本 skill 目录后：`python3 -m pytest tests/ -v` 应全绿且 ≥35 例（当前 93 例）。
另外每次真实跑完必须补一句证据：`verify_sources.py verify` 退出码为 0。

## 6. 已知边界（不许对她夸大）

- **预存回款（约 29%）拿不到 SO**：智云核销明细子表接口不返回订单号，需要与「下单栏」按金额配对才能还原，
  而「下单栏」尚未纳入数据契约 → 这类目前进 hold。**不得声称三通道全部打通。**
- **手续费口径未定**：实测到账净额 298.38 + 手续费 1.62，系统核销记 300.00。
  匹配时两种口径都试，但**回填金额一律挂起**等她拍板。
- **尾差阈值未定**：实测见过 0.12 / 0.32。默认阈值 0（不容忍、挂起），不许自己设。
- **一个 SO 占多行**（实测 14.5%）：一律列候选行让她选，不默认取某一行。
