# 应收核销日清（人在环 · SOD 级判定 · 回填硬闸）

**主战场从第 6 步开始**（明妹钉死）：智云按核销日筛 → 判定 → **《核销日清》** → 她确认 → 写工作副本「明细」。
1–4 她提才做；第 5 步销售不绑隔天。

**取数只有一个入口**：财务管理 →「回款记录」，**只筛核销日期 = T-1 一个条件**，
再抓每笔点进去的明细页（下单 / 同币种核销明细 / 由 SO 查订单明细拿 SOD）。
**「回款核销对账」表已废弃**——它对预存/预收 0 行、拿不到 SOD，2026-07-22 静默丢 3 笔到账就出在它身上。

**数据源**：第 6 步智云 = 核销真相；第 7 步流转表 = 备份 + 写单号 +「是否更新」状态，**回填金额不从第 7 推**。
**写表**：只动盈亏「明细」；其它 sheet 禁止。
**永不写智云。** 金额只由 `scripts/*.py` 算。

## 0. 按她的话直接跳

|---|---|---|
| 跑昨天的核销 / 出清单 / 日清 | **取数→判定→校验→日清清单，然后停** | `fetch_zhiyun`(可选) → `classify_hexiao` → `validate_plan` → `build_worklist` |
| 确认 / OK / 可以写 / 按这个写 | **仅在有日清清单后** `apply --confirmed` | 见 §3.1 |

## 0.5 硬禁止

1. 禁止自写 Python 读她的 Excel；只准调 `scripts/`。
2. 禁止把客户名、金额明细打进对话。
3. 缺输入就停下报缺，禁止凑合（`load_exports` 缺件会直接非 0 退出并指路）。
4. **禁止跳过《核销日清》直接 `apply_to_copy`**（脚本无 `--confirmed` 也会拒绝）。
5. 禁止自动写到账流转表（只出建议）。
6. **禁止把「判不出来」悄悄跳过**：每笔到账必须产出判定，覆盖率校验不过就报错退出。

## 1. 红线

1. 金额只由脚本算。
2. 对话不回显客户名金额。
3. 分笔一律 hold，禁止均分/比例分摊。
4. 永不写智云；账密不落盘（`fetch_zhiyun` 运行时 getpass，用完即丢）。
5. 写盈亏只许「明细」；`--in-place` 写前备份、写后回读，不过不替换。
6. **回填 = 日清清单 → 她确认 → `apply --confirmed`**。
7. 回填数据源 = 第 6 步智云，不用流转表金额。
8. 列名模糊匹配；找不到列 → 报错列实际表头。
9. **AR 覆盖率硬校验**：判定覆盖的到账集合必须等于回款记录集合，不等就 `CoverageError` 退出。

## 3. 流程编排（11 步）

| 步 | 名称 | 谁做 | 入口 |

| 5 | 销售核销 | 销售 | 不绑隔天 |

### 3.1 回填硬闸（fetch → classify → validate → 清单 → confirm → execute）

```bash
SKILL="<本skill目录>"
WS="$SKILL/工作区"
LEDGER="$WS/02_我的表副本/<盈亏副本.xlsx>"   # 她固定用的那份

python3 "$SKILL/scripts/fetch_zhiyun.py"    --date yesterday --workspace "$WS"
python3 "$SKILL/scripts/verify_sources.py"  snapshot --workspace "$WS"
python3 "$SKILL/scripts/inspect_inputs.py"  --workspace "$WS"
python3 "$SKILL/scripts/classify_hexiao.py" --workspace "$WS"
python3 "$SKILL/scripts/validate_plan.py" \
  --plan "$WS/04_产出/判定结果_*.json" --ledger "$LEDGER" \
  --out "$WS/04_产出/写入计划_校验后.json"
python3 "$SKILL/scripts/build_worklist.py"  --workspace "$WS"
# ★ 到这里必须停。把《核销日清》路径给她。禁止继续 apply。

# 仅当她说「确认/OK/可以写/按这个写」：
python3 "$SKILL/scripts/apply_to_copy.py" \
  --checked "$WS/04_产出/写入计划_校验后.json" \
  --ledger "$LEDGER" --confirmed --in-place
# 头几次现场可去掉 --in-place，出新文件并排验

python3 "$SKILL/scripts/verify_sources.py" verify --workspace "$WS"
```

确认词（同义）：`确认` `OK` `ok` `可以写` `按这个写` `没问题写吧` `写吧`。
她说改/跳过某笔 → 先改计划或加 `--force` 只写无冲突的，**重新出清单**再确认。

### 3.2 流转「是否更新」三态（只建议）

| 状态 | 填法 | 公式 | 颜色 |
|------|------|------|------|
| 全部订单都没检索到/不能更 | **留空** | **不要公式** | — |
| 全部检索到并已更新 | **是** | 可套她惯用公式，否则字面「是」 | — |
| 部分更新 | **部分** | 不要写成「是」 | **未更新 SO 标红** |

实现：`flow_ledger.flow_status_policy`；清单页签「流转表怎么填」。

### 开关

- `--flow-complete`：当天**所有渠道**流转表都给了才加（否则不误判 E0）。
- `--ledger`（rescan）：给了才对挂起项真重判。
- `--confirmed`（apply）：**人工确认后才写**。
- `--in-place`（apply）：就地写同一份副本（写前备份）。

规则 → `config/业务规则.md`、`references/规则_取数与判定.md`、`references/规则_判定决策树.md`；
给她看 → `references/使用说明_给明妹.md`。

## 4. 你（agent）怎么干

1. 盘点输入；缺料一次说清。
2. 主路径跑到 **build_worklist** 停下，用 §0 照抄结尾。
3. 等确认词 → 再 `apply --confirmed`。
4. 对话只报笔数/路径/成败，不报明细。
5. 依赖：`pip install openpyxl xlrd`；智云另需 requests/playwright。
6. 测试：`python3 -m pytest tests/ -v`（应 ≥130 例全绿）。

## 5. 验收

- pytest 全绿（含真实金标端到端，本地有测试数据时跑）。
- 无 `--confirmed` 时 apply 必须非 0。
- 真跑后 `verify_sources.py verify` 退出 0（未确认前源表哈希不变）。
- **金标基线（2026-07-22 真实 13 笔）**：157 个 SOD 行、可填 143、与她手工填的**逐格一致 143/143**、
  异常 0、13 笔到账全覆盖。改完代码这条必须仍成立。

## 6. 已知边界（不许夸大）

- 分笔 / 手续费 / 尾差 / SOD 凑不出唯一组合 / 跨年：未拍板一律 hold。
- 流转表**不自动写**；红字字体 openpyxl 难稳读。
- 她公式的具体式子以她表为准，程序给策略不发明公式。
- **拆行场景**（她把一个 SOD 拆成「已收/未收」两行）程序只提示、不代拆；
  且她在拆出来的行上**计提留空**的原因尚未问清 → 这类一律 E5 挂起，不猜。
