# 销售归属维护表

> **这是一张「活」表，由人维护。** 每周可能有人离职、客户转手。
> skill 每次跑应收时会**读这张表**，把"这次准备把谁的账归给谁"摆出来让你**确认**；
> 你说"确认"就执行，要改就**直接改这张表**、重跑、再确认，改完 `git commit` 留痕。
> 归属不由脚本/AI 私自拍板——一切以本表 + 你的确认为准。

## 段一 · 按销售接手（离职销售名下的账，默认归接手人）

- **处理方式**（只决定**最终名字怎么写**）：
  - `细分追溯` = 用复合名「接手-离职」保留痕迹（如 `梁玲玲-高美杰`），用于遗留坏账要追溯来源的人。
  - `直接接手` = 直接换成接手人名字（光名，如 `王雄`）。

## 段二 · 按公司接手（特定客户整体转给指定人）

- 客户名用**关键词包含**匹配（如「华电」可匹配「中国华电科工集团有限公司」）。**键越具体越好**（用全名、少用太短的词）；**具体键自动盖过笼统键**（按长度降序匹配）。
- **范围列（多周成品实证的关键）**：
  - `仅高美杰` = 这条客户规则**只对高美杰名下的行生效**（华电/特变/中国机械都是"高美杰遗留客户"）。所以**骆利飞名下的特变不被它抢走**、落到段一→王雄；高美杰名下的特变→于占国-高美杰。
  - `通用`（默认/留空）= 对**所有行**生效（含在职、含离职如骆利飞）。如中国国际电视/芒果/广州库洛——骆利飞名下的这些单也跟着走。
- **优先级**：先按段二客户规则（命中且范围适用就归指定人），没命中才落段一（离职默认接手人）。即"**先看客户、客户没说话再看人**"。
- 维护提示：某个大客户整体换人负责，就在这里加一行；拿不准范围就填 `通用`（只有"专门给高美杰遗留客户兜底"那种才填 `仅高美杰`）。
