# 销售归属维护表

> **这是一张「活」表，由人维护。** 每周可能有人离职、客户转手。
> skill 每次跑应收时会**读这张表**，把"这次准备把谁的账归给谁"摆出来让你**确认**；
> 你说"确认"就执行，要改就**直接改这张表**、重跑、再确认，改完 `git commit` 留痕。
> 归属不由脚本/AI 私自拍板——一切以本表 + 你的确认为准。

## 段一 · 按销售接手（离职销售名下的账，默认归接手人）

- **处理方式**（只决定**最终名字怎么写**）：
  - `细分追溯` = 用复合名「接手-离职」保留痕迹（如 `梁玲玲-高美杰`），用于遗留坏账要追溯来源的人。
  - `直接接手` = 直接换成接手人名字（光名，如 `王雄`）。
- 维护提示：有人离职就在这里加一行；名字**务必和源台账里的写法一致**（错别字会对不上，程序会把疑似错别字列进"归属存疑"让你确认）。

## 段二 · 按公司接手（特定客户整体转给指定人）

- 客户名用**关键词包含**匹配（如「华电」可匹配「中国华电科工集团有限公司」）。
- **对所有行生效、优先级高于段一**：这些客户整体归指定人——不管现在谁持有（含在职销售），客户命中就归这里指定的人。
- **键越具体越好**（用全名、别用太短的词）：短键如「特变」会命中所有含"特变"的客户，将来万一来个不相干的同名客户会被错误归走。**具体键会自动盖过笼统键**——所以"特变电工新疆新能源股份有限公司→王雄"这种**会优先于**"特变→于占国"，需要细分就加具体那条。
- 维护提示：某个大客户整体换人负责，就在这里加一行。
