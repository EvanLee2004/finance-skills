## 三、确定性规则（写死在脚本里，这里只解释，便于核对）

这些口径稳定、不该改，列在这里是为了**让人看懂脚本在干嘛、能核对**：

- **账龄(月份)** = (基准年 − 交付年)×12 + (基准月 − 交付月) − **1**。基准月 = 运行当月或源文件名里的月份。
- **删 0 行**：应收金额 = 0 的行删掉（客户已回款 / 已核销）。
- **结转老坏账**：从上一版 all 里，把"**年度早于源表最早年份**（如源表只有 2019+，则 2016/2017/2018）、**且应收≠0**"的老账结转进来——这些是源台账已不再列、但还没回款的历史账；近期年份不在源表 = 已结清，不结转。
- **新智云单号补位**：该列为空时，用"单号/老单号"补上（它是回填和结转的钥匙）。
- **列对齐**：源表各列 → 统一 17 列，靠 `列名别名.json` 认列；**认不出的关键列会被脚本大声告警**（不会静默丢）。
- **销售归属**：见 `销售归属维护表.md`（人维护、每次让人确认）。
