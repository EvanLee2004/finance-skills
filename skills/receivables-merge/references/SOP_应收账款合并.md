# SOP · 应收账款合并（赵 Excel 流程）

> 定稿：2026-06-18，李明昊。**依据**：赵手工操作视频（11分36秒）逐帧 + 真实成品文件 `2026.6.4应收all.xlsx` 逐列逐行反推，零推测。
> 旧流程（含 `merge_receivables.py`）相比本 SOP **漏了三步**：S5 删应收=0行、S4 维护表驱动归属、S8 透视汇总。本 SOP 为现行准。

---

## 输入（3 个文件）

| 文件 | 说明 |
|------|------|
| **源台账**（如 `2026.6.4日应收.xlsx`） | 本周原始，按年份分 sheet（2026/2025/2024/6月批量/2023…2019），各 sheet 表头不统一 |
| **回填源**（上一版 `应收all`） | 上周做好的总表，带销售手填标注列 |
| **维护表**（`营销人员应收匹配规则.xlsx`） | 销售变化表：①离职销售→接手销售 ②客户需重新分配→接手销售 |

## 输出

`应收all.xlsx`：① 17 列干净主表 ② `Sheet2` 透视汇总（应收金额 按 销售人员→客户）

**17 列固定顺序**：年度 · 销售人员 · 客户名称 · 新智云单号 · 文件名 · 应收金额 · 交付月份 · 账龄(月份) · 结算阶段 · {日期}销售预计回款日期 · 销售解释说明 · 有无合同 · 合同分类 · 框架合同是否存在PO单或下单记录 · 应收金额是否有客户正式确认 · 客户结算周期 · 是否按月给客户发结算单
（**无独立"单号"列、无"翻译类型"列**）

---

## 操作步骤

### S1 · 合并各 sheet 成一张主表
对源台账每个 sheet：
1. **打年度标签**：新增"年度"列，整列填 = 该 sheet 名（2024 sheet→2024）；**`6月批量`→填"技术统一确认"，排在最末尾**。
2. **补全新智云单号（回填的钥匙）**：有新智云单号就用它；**只有单号/订单号的，把它复制成新智云单号**（覆盖 2024 空号行、2023 及更早无此列的年份、6月批量用订单号）。
3. **按列名对齐到 17 列骨架**（**按列名认，不按列位置**）：
   - **应收金额**：优先级 `应收金额 ＞ 订单折合本币(6月批量) ＞ 金额`
   - **交付月份**：先取现成年月列 `交付月份/项目交付/销售确认`；都没有 → 从日期列 `完成时间/项目交付日期/交件日期/客户交付日期/截止时间/下单时间` 推出 YYYYMM
   - 其余列（销售人员、客户名称、文件名）按名对应；**丢弃 单号、翻译类型、字数、价格、逾期 等多余列；6月批量 19 列只留映射得上的几列**
4. 各 sheet 数据纵向追加，堆成一张大表。

### S2 · 算账龄(月份)
- 公式口径：`账龄 =(LEFT(基准月,4)−LEFT(交付月,4))×12 +(RIGHT(基准月,2)−RIGHT(交付月,2)) − 1`
- 从 YYYYMM 文本用 LEFT 取年、RIGHT 取月；**末尾 −1 不能漏**。
- **基准月 = 运行当月**（自动取电脑当月，如 202606）。
- 算完把账龄**取值固化**，删临时辅助列。

### S3 · 回填标注列
- **以"新智云单号"为键**，从**回填源** `VLOOKUP` 逐列回填：结算阶段 / 预计回款日期 / 销售解释说明 / 有无合同 / 合同分类 / 框架合同PO / 客户确认 / 客户结算周期 / 是否按月发结算单。
- 回填源表头**日期前缀每周漂移**（如 `0508→0528 销售预计回款日期`）→ **按关键词认列**，别死扣全名。
- 回填后**粘成值**固化公式。
- 早年份无新智云单号 → 部分 `#N/A` 属**正常**，列入复核清单。

### S4 · 销售归属（维护表驱动）★旧流程没有
对每一行，定最终"销售人员"，**按优先级**：
1. **客户重分配优先**：若"客户名称" **contains** 维护表"客户需重新分配"某键 → 接手 = 该键对应接手销售。
   - 例（实证）：客户含`华电`/`特变`→`于占国`；客户含`中国机械设备工程`→`高美杰1`。
2. 否则，若原销售 ∈ 维护表"离职销售" → 接手 = 其默认接手销售（如 `高美杰→梁玲玲`）。
3. 否则 → 原样不动。
4. **命名格式**：最终销售 = **「接手-原销售」复合名**（如 `于占国-高美杰`、`梁玲玲-高美杰`）；
   **例外**：接手值本身是坏账桶码（如 `高美杰1`，已含原销售名）→ 用**光名**，不加后缀。

> 实证（成品文件）：高美杰 188 行分三路 —— `高美杰1`×10（全=中国机械设备工程）、`于占国-高美杰`×74（全=华电/特变系）、`梁玲玲-高美杰`×104（其余）。**客户优先级 > 离职默认** 由此确认。

### S5 · 删行（已回款/核销）★旧流程没有
- **删掉"应收金额 = 0"的行**（客户已回款 / 已核销，应收清零）。
- 实证：源表 175 个应收=0 行，成品里此类行 = **0**。

### S6 · 复核
- **年度自检**：派生年度 vs 交付月份里的年份，不符 → 告警（防 2020 误标 2021 那类人为笔误）。
- **#N/A 清单**：回填查不到的单号单独列出，交人工。
- **名称残留检查**：成品里不应再出现"离职销售光名"（应已全部归属）。

### S7 · 排序
- 按 销售人员 / 年度 等维度排序，便于阅读与分块复核。

### S8 · 汇总透视 ★旧流程没有
- 生成 `Sheet2` 透视表：**应收金额 按 销售人员 → 客户** 求和（看各销售、各客户应收总额）。

---

## 会变 vs 固定（给自动化划界）

- **固定逻辑**（写死）：年度标签、新智云单号补位、账龄公式、VLOOKUP 回填、删 0 行、年度自检。
- **会变·靠配置表**：① 列名别名（S1 认列）→ 别名表；② 销售归属（S4）→ 维护表。规则变 = 改表，不改码。
- **遇异常停下问人**：出现别名表里没有的新列名、或归属拿不准 → 不瞎来，报人工。
