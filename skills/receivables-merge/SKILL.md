# 应收账款合并（赵 Excel 流程）

把财务部每周手工做的"应收账款合并"自动化。完整流程与口径依据见 `references/SOP_应收账款合并.md`（**动手前先读它**），设计思路见 `references/思路_skills化.md`。

## 你（agent）该怎么干

**核心原则：人说人话，找文件/认文件/跑脚本/出复核 全归你，别让用户改文件名、摆文件夹。**

### 1. 找齐三个输入（按内容认，不靠文件名）
需要：①**源台账**（本周原始，多个年份 sheet）②**回填源**（上一版做好的 all，带销售标注列；**还用于结转老坏账**）③**维护表**（`config/销售归属维护表.md`，两段：离职→接手 / 客户→接手；默认已在 `config/`）。

先跑识别看认得对不对：
```bash
python3 scripts/merge.py --inspect --input-dir <放文件的目录>
```
- 默认去 `工作区/input/` 找；用户的文件在别处就把 `--input-dir` 指过去。
- 认出来后**跟用户核一句**"源台账=X、回填源=Y，对吗"。**认不准 / 缺文件 / 有歧义 → 问用户，不要猜。**

### 2. 跑合并
```bash
python3 scripts/merge.py --source <源台账> --ref <回填源> --rules config/销售归属维护表.md --out 工作区/output/应收all_<日期>.xlsx
```
- `--ref`（上一版 all）省略则跳过回填**和老坏账结转**（标注列留空、缺 2016-2018 老账）；`--rules` 省略则跳过销售归属。**要完整复现成品，必须给 `--ref`。**
- 账龄基准月默认取源文件名里的年月、再退电脑当月；补往月报告用 `--base-month 202606`。
- 脚本按 SOP 跑 S1–S8：合并 → 账龄 → 回填 → **销售归属（维护表驱动）** → **删应收=0行** → 复核 → 排序 → **透视汇总**。

### 3. ⭐ 确认销售归属（每次必做，不许跳过）
**销售变更是「会变、要人确认」的事，绝不能由脚本/你私自拍板。** 跑完后：
1. 打开产物里的 `归属变更`（这次准备把谁→归给谁、各多少行）和 `归属存疑`（疑似错别字/离职名对不齐，如「牛雪如 vs 牛雪茹」）两个 sheet。
2. **明确摆给用户**："这次按销售维护表，准备这样归属（列出变更）；另有这些存疑（列出）。**确认还是要改？**"
3. **用户说"确认"** → 就照这个定稿，往下走。
4. **用户说要改**（如"骆利飞归王雄""牛雪如就是牛雪茹"）→ 按用户说的**改 `config/销售归属维护表.md`**（加人/改名/调接手）→ **重跑** → 再确认。
5. **改完把改动固化**：`git add config/销售归属维护表.md && git commit`，让这次的销售变更留痕、下次自动生效。

> 维护表是"活"的：每周可能有人离职/客户转手。**你的职责是把变更和存疑摆清楚、让人一句话拍板，然后忠实落到表里**——不是自己猜规则、不是写死在代码里。

### 4. 交付其余复核结果
产物还含：`未匹配复核(#N/A)`、`离职残留`（应为空）、`被删0行`（已回款留底可追溯）、`年度校验`、`透视汇总`、`运行报告`。**主动提醒用户重点看**未匹配与被删0行。

## 会变的东西在哪
- **列名别名**（认列规则）→ `config/列名别名.json`
- **销售归属**（离职→接手、客户→重分配）→ `config/销售归属维护表.md`（用户维护）
规则变了改这两张表，**不用改代码**。

## 验收口径（判断脚本对不对）
用真实历史数据跑，主表的行数 / 销售人员分布 / 高美杰三路 / 应收=0行数，应能对上已知正确成品。详见 SOP「验收标准」。
