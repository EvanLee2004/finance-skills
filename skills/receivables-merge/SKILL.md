# 应收账款合并（赵 Excel 流程）

把财务部每周手工做的"应收账款合并"自动化。完整流程与口径依据见 `references/SOP_应收账款合并.md`（**动手前先读它**），设计思路见 `references/思路_skills化.md`。

## 你（agent）该怎么干

**核心原则：用它的是财务同事（如赵亮晶），不是技术员。** 她扔个 Excel 进来，你把活办完——找文件 / 认文件 / 跑脚本 / 出复核全归你；**别让她改文件名、摆文件夹、看命令、看 sheet 名**。

### ⭐ 人在环：什么自己干、什么问她（这是这个 skill 的灵魂，拿捏好分寸）

> 原则：**确定的、可逆的、常规的、她也答不上来的 → 自己干**；**只有她才知道的、有歧义的、对外发出去的产物 → 问她**。问要：**批量问（别一条条）、用人话（别甩告警/sheet）、给摘要（别甩原始数据）**。问太多=橡皮图章，她会烦、会乱点。

**自己干（别问，问了烦人）**
- 跑脚本、算账龄、删已回款/小额(|应收|≤1)行、按销售→客户排序、出透视——确定性、可复现的活。
- 认文件：能按内容认出源台账/回填源就**直接认**，**认对了只一句话告知**（"我把 X 当本周源表、Y 当上一版 all 了"），不用她逐个确认。
- 命令、列名映射、sheet 名、参数——你消化，绝不甩给她。

**问她（克制！正常的一周只会打扰她 1 次，别造成"同意疲劳"）**
1. **唯一固定要问的一句：跑之前轻问销售有没有变动**（她才知道，组织每周可能变）：
   *"跑这周应收前确认下：销售有谁离职 / 新接手、大客户换人负责吗？没有就回'没变'（不回我也默认没变照跑）。"*
   —— 就这一句、她能秒回甚至不回。**别再为别的常规事拦她。** 文件**有歧义**时才额外让她认一下哪个是源表/哪个是 all（不歧义就只告知、不用她点头）。
2. **只在"出异常"时才再拦她**（出异常才拦截，没异常走快车道）：
   - 表变了（认列告警）→ **人话问**："你这周表里'应收金额'好像改叫'应收款'了，是同一个吗？" / "多了个'9月批量'的 sheet，算哪年、整批按哪个月算账龄？"
3. **跑完按"有没有值得她看的"决定怎么收尾**：
   - **常规（0 存疑、没新面孔、销售没变）→ 一句话带过**："跑完了，3243 行，归属按现有维护表走的、没异常，表在 <路径>。" **别为没变化的归属也逼她确认。**
   - **有存疑/新面孔/大变动 → 才重点用人话摘要问**："这次把高美杰分给梁玲玲/于占国、骆利飞→王雄…（关键几条）；有 X 处拿不准（如疑似把'牛雪如'当'牛雪茹'），对吗、要改吗？"

### 1. 找齐三个输入（按内容认，不靠文件名）
需要：①**源台账**（本周原始，多个年份 sheet）②**回填源**（上一版做好的 all，带销售标注列；**还用于结转老坏账**）③**维护表**（`config/销售归属维护表.md`，两段：离职→接手 / 客户→接手；默认已在 `config/`）。

⚠️ **路径全用绝对路径，别 cd、别硬编码别人机器的路径。** 本 skill 的脚本/配置就在**「这份 SKILL.md 所在的文件夹」**里——你读到这份文件，就知道它的绝对路径，**下面用 `<本skill目录>` 代指它，调用时换成真实绝对路径**。脚本会按自身位置自动找 `config/`，所以**别手动传 `--rules`**。

```bash
# 把 <本skill目录> 换成这份 SKILL.md 实际所在文件夹的绝对路径
python3 "<本skill目录>/scripts/merge.py" --inspect --input-dir <放文件的绝对目录>
```
- 默认找 `工作区/input/`；用户文件在别处就把 `--input-dir` 指过去（**绝对路径**）。
- 认出来后：**不歧义就一句话告知**（"源台账=X、回填源=Y，我按这个跑了"）；**有歧义 / 认不准 / 缺文件才让她确认文件**。（销售变动那句无论如何都轻问一下，见人在环①——它和认文件是两件事。）

### 2. 跑合并（全绝对路径、别传 --rules、别写死 base-month）
```bash
python3 "<本skill目录>/scripts/merge.py" \
  --source <源台账绝对路径> --ref <回填源绝对路径> \
  --out <交付目录绝对路径>/应收all_<日期>.xlsx
```
- **别传 `--rules`** → 自动用 skill 自带 `config/销售归属维护表.md`（按脚本自身位置解析，跨机器都对）。
- **账龄基准月不用管**——脚本自动从源文件名取（`2026.6.4日应收` → 202606）。**只有补往月报告**才显式加 `--base-month 202606`（**别照抄这个值到别的月份，会把账龄算错**）。
- `--ref`（上一版 all）省略则跳过回填**和老坏账结转**。**要完整结果必须给。**
- **`--out` 可省**：省了默认就落在**源文件同目录**（她在哪放的源表就在哪拿，找得到）；想放别处就用绝对路径 `--out`。脚本会自动建目录。**无论哪种，跑完把最终文件路径明确告诉她。**
- 脚本按 SOP 跑 S1–S8：合并 → 账龄 → 回填 → **销售归属** → **结转老坏账**(账龄重算+归属重算) → **删小额行(|应收|≤1)** → 复核 → **排序(销售→客户)** → **透视汇总**。

### 3. ⭐ 销售归属：自己先看清，按"值不值得她看"决定深浅
**销售变更绝不能由脚本/你私自拍板，但也别为没变化的归属逼她确认。** 跑完：
1. **你自己**看产物里的 `归属变更`（谁→归给谁、各多少行）和 `归属存疑`（疑似错别字/对不齐的名）两个 sheet——**这是给你看的，不是给她看的**。
2. **按情况收尾**（同人在环③）：
   - **0 存疑、没新面孔 → 一句话带过**，不用她拍板。
   - **有存疑 / 有维护表里没有的新销售名 → 才人话摘要问她**："…（关键几条 + 存疑），对吗、要改吗？"
3. **她口头说要改，你就直接编辑 MD 维护表**（这就是"活表"的用法，别让她去碰 Excel/代码）。听懂意思 → 改对应的表 → **重跑** → 把结果给她确认 → **`git add config/ && git commit` 固化**（下次自动生效、留痕）：
   - 离职 / 接手 / 改名（"X 离职了账归 Y""牛雪如就是牛雪茹"）→ 改 `config/销售归属维护表.md` **段一**（离职|接手|处理方式，加行或改行；要保留来源痕迹用 `细分追溯`，否则 `直接接手`）。
   - 某客户整体换人（"广州库洛以后归赵贺斌""中国国际电视归于占国"）→ 改 **段二**（加一行：客户关键词|接手人|范围）。**范围一般填 `通用`**（对所有人名下该客户都生效）；只有"专门给高美杰遗留客户兜底、不许动别人名下同名单"才填 `仅高美杰`。
   - 改小额阈值 / 特殊批次 → 改 `config/业务规则.md`。

> 维护表是"活"的：每周可能有人离职/客户转手。**你的职责是把变更和存疑摆清楚、让人一句话拍板（或听她口头指令），然后忠实落到 MD 表里**——不是自己猜规则、不是写死在代码里。

### 3.5 ⭐ 像同事一样收尾（数字员工三件事 · 主动但克制，和上面人在环一脉相承）
跑完别只报"X 行、在哪"，像个做了多周应收的老同事那样多看一眼：
1. **环比/异常小结**：和上一版 all（你手里就有 `--ref`）比一句——"这周行数 X（上版 Y，**多/少了 N**）；**新出现的销售名**有谁（维护表没登记的）；归属大变动的客户"。本次内部也扫："有没有金额异常大的单、#N/A 突然变多"。**没异常就一句话带过**，有才展开。
2. **拿不准就提具体问题、别瞎猜**：疑似错别字名、维护表没覆盖的新销售、未知 sheet——提一句能秒回的（"'牛雪如'是不是就是维护表里的'牛雪茹'？""多了个销售'王某'，维护表没有，他是新接手谁的？"），别自己拍板也别默默放过。
3. **本岗位该主动操心啥**（老员工的留心眼）：
   - **行数和上周对不上**（突然多/少几百行）→ 八成源表漏放了某年 sheet 或多放了，提醒她。
   - **离职残留 sheet 不为空** → 有离职销售的单没归出去，维护表可能缺人。
   - **#N/A（未匹配）突然变多** → 主键/回填源可能不对版。
   - **归属存疑** → 疑似对不齐的名字，宁可问一句。

### 4. ⭐ 认列告警 → 用人话问她（表变了的早期信号）
**跑完你先看 `认列告警` sheet（给你看的，运行日志也会打 ⚠）**——这是她表变了的关键信号：
- **「关键列没认出」**：她表里某列改了名（如"应收金额"→"应收款"），这批会**丢失/算错**。→ **用人话问她**（见人在环②）确认是不是同一个意思，确认后把新列名加进 `config/列名别名.json`，重跑。
- **「未知 sheet」**：多了个不是年份、没登记的 sheet（如"9月批量"）。→ **别猜**，人话问她这是不是新批次、记哪年/哪月，确认后补进 `config/业务规则.md`，重跑。

其余产物（`未匹配复核(#N/A)`、`离职残留`应为空、`被删0行`留底、`透视汇总`、`运行报告`）**你自己核**；只有"被删的是不是真已回款""#N/A 那几单要不要补数据"这类**她才知道的**才提她一句。别把一堆 sheet 全甩给她。

## 会变的东西在哪（都是活的，改表不改码）
- **列名别名**（认列规则）→ `config/列名别名.json`
- **销售归属**（离职→接手、客户→重分配）→ `config/销售归属维护表.md`（用户维护、每次确认）
- **特殊批次 / 跳过的 sheet / 业务规则说明** → `config/业务规则.md`（agent 跑前先读；遇表里没覆盖的情况停下问人）
确定性算术（账龄、合并、删0、结转、求和）写死在脚本里保证可复现；**会变/要判断的都在上面这几张 MD/表里**。

## 验收口径（判断脚本对不对）
**已知基准**（源台账 `2026.6.4日应收` + 回填源 `2026.5.28应收all`，`--base-month 202606`）：
- 主表 **3243 行**（已与成品行数一致）；高美杰三路 **梁玲玲-高美杰 104 / 于占国-高美杰 74 / 高美杰1 10**；删小额行(|应收|≤1) **179**；结转老坏账 **29**；离职残留 **0**。
- ⚠ **多周成品(5.14/5.21/5.28/6.4)实测**：每周行数全部精确吻合；销售归属除「广州库洛 郑瑞→赵贺斌」(亮晶姐 0620 的修正、旧成品还没改) 外**逐行一致**（5.28 周 0 差异）。结转老账的账龄现按当期重算（不再等于旧成品冻结值）。**这条广州库洛差异是对的。**
- 口径来源：亮晶姐 0620 答复 + 多周成品实证。已落地：删≤1、结转账龄每月涨+销售名重算、排序销售→客户、广州库洛/网易→赵贺斌、**骆利飞名下跟客户走**（中国国际电视/芒果/内蒙古→于占国）唯特变→王雄，华电/特变/中国机械等"高美杰遗留客户"只对高美杰名下生效（维护表段二"范围=仅高美杰"）。见 `20260618_…-已回复.docx`。

换别的周数据跑：看 `认列告警`（应为空）、`离职残留`（应为空）、年度分布合理、不崩，即结构稳。
