# 应收账款合并（赵 Excel 流程）

把财务部每周手工做的"应收账款合并"自动化。完整流程与口径依据见 `references/SOP_应收账款合并.md`（**动手前先读它**），设计思路见 `references/思路_skills化.md`。

## 你（agent）该怎么干

**核心原则：人说人话，找文件/认文件/跑脚本/出复核 全归你，别让用户改文件名、摆文件夹。**

### 1. 找齐三个输入（按内容认，不靠文件名）
需要：①**源台账**（本周原始，多个年份 sheet）②**回填源**（上一版做好的 all，带销售标注列）③**维护表**（`营销人员应收匹配规则.xlsx`，含离职/接手列；默认已在 `config/`）。

先跑识别看认得对不对：
```bash
python3 scripts/merge.py --inspect --input-dir <放文件的目录>
```
- 默认去 `工作区/input/` 找；用户的文件在别处就把 `--input-dir` 指过去。
- 认出来后**跟用户核一句**"源台账=X、回填源=Y，对吗"。**认不准 / 缺文件 / 有歧义 → 问用户，不要猜。**

### 2. 跑合并
```bash
python3 scripts/merge.py --source <源台账> --ref <回填源> --rules config/营销人员应收匹配规则.xlsx --out 工作区/output/应收all_<日期>.xlsx
```
- `--ref` 省略则跳过回填（标注列留空）；`--rules` 省略则跳过销售归属。
- 账龄基准月默认取源文件名里的年月、再退电脑当月；补往月报告用 `--base-month 202606`。
- 脚本按 SOP 跑 S1–S8：合并 → 账龄 → 回填 → **销售归属（维护表驱动）** → **删应收=0行** → 复核 → 排序 → **透视汇总**。

### 3. 把复核结果交给人
产物 `应收all` 含多个 sheet：`主表`、`透视汇总`、`未匹配复核(#N/A)`、`名称变体`、`离职残留`、`年度校验`、`被删0行`、`运行报告`。
**主动提醒用户重点看**：`未匹配复核`、`离职残留`（应为空）、`被删0行`（留底可追溯）。有异常或拿不准的归属 → 报给用户定，别擅自改。

## 会变的东西在哪
- **列名别名**（认列规则）→ `config/列名别名.json`
- **销售归属**（离职→接手、客户→重分配）→ `config/营销人员应收匹配规则.xlsx`（用户维护）
规则变了改这两张表，**不用改代码**。

## 验收口径（判断脚本对不对）
用真实历史数据跑，主表的行数 / 销售人员分布 / 高美杰三路 / 应收=0行数，应能对上已知正确成品。详见 SOP「验收标准」。
