# 九点下单统计 · 业务规则

## 日期窗口 `date_window(today)`

| 运行日 weekday | 抓取下单日期 |
|----------------|--------------|
| 周一 (0) | 上周五～上周日 |
| 周二～周五 (1–4) | 昨天～昨天 |
| 周六/日 (5–6) | 上周五～昨天（若触发；可用 `--force`） |

## 组织架构

- 源：`config/销售组织架构.xlsx` sheet「组织架构」
- A 列 = 销售姓名，B 列 ∈ {本地化, 数据, 游戏, 其他}
- **展示名**：`本地化` → **`多语（不含运保）`**
- 未匹配销售：金额进列 **「（未匹配）」**，**不**并入四部门合计；名单写处理日志

## 金额

- 优先字段：`下单预估额/本币`
- 回退：`下单预估额/原币`（日志写实际字段）
- 万元 = 本币 / 10000；按 **日期 × 展示部门** 求和；输出 **round(, 2)**

## 产物版式

sheet「下单数据」：行=日期，列=总计 | 多语（不含运保）| 数据 | 游戏 | 其他 | （未匹配）  
sheet「处理日志」：窗口、行数、总计、金额字段、未匹配名单  
`--detail`：同簿 sheet「明细」

## 非目标（v1）

- 不回写亮晶全年累计表指定单元格
- 不替代利润看板；不使用智云「部门」字段代替组织架构
- 不抓回款/付款等其他表
