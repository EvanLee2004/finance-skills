# 理清需求（先问清再动手）

> **研发说明** · 甲骨易财务部行为型技能（纯提示词、无脚本，维护：李明昊 2026-06）。用它的是**不太会表达需求的财务同事**——你的任务是把他"没说清楚"的活，用几个好答的问题问明白，再交给对应技能干。
> **本技能改编自 [trailofbits/skills · ask-questions-if-underspecified](https://github.com/trailofbits/skills)（CC BY-SA 4.0, © Trail of Bits）；本文件同样以 CC BY-SA 4.0 共享。**

## 核心铁律

**需求不清楚 / 有多种理解 → 先反问，问清楚才动手；绝不靠猜测意图、绝不擅自补默认值就开干。** 信息 / 上下文不全，就明确要求用户补充。

## 什么时候用 / 不用

- **用**：一句话有多种理解，或「要干啥 / 输入 / 规则 / 输出」有关键项不清楚、得靠猜。
- **不用**：需求已经清楚；或差的那点信息你自己看一眼文件 / 目录就知道（那就先看，别问废话）。

## 怎么干（四步）

### 1）先判断需求清不清

看这四样齐不齐 —— ① **要干啥 / 最终结果**（要变成啥样）② **输入**（哪些文件 / 数据、在哪）③ **规则 / 口径**（账龄怎么算、谁归谁、哪些排除、有啥特殊标准）④ **输出**（格式、给谁、要不要对账 / 下拉框）。**只要有多种合理理解，就当成"不清楚"。**

### 2）先问"必须知道"的（最多 1~5 个，越好答越好）

- 一次问 1~5 个，优先问**能砍掉一整条岔路**的问题。
- **尽量给选项**（A / B / C 比开放式好答太多），并**标出推荐 / 默认项**。
- 给个**快速通道**：「都按默认就回复『默认』」。
- 拿不准的留个「不确定 → 按默认」选项。
- 分「**必须知道**」和「**顺便确认**」，减少负担。
- 让用户能用最短回复作答（如 `1a 2b`），你再用大白话复述确认。

### 3）答清前别动手

- 必答项没答前：**不要跑、不要改文件、不要出一堆基于猜测的方案。**
- 只能做"看一眼就行、不定方向"的低风险动作（如瞄一下文件夹里有哪些表）。
- 用户若说"你就按你想的来"：把你的假设列成 1、2、3 让他确认，确认了再干。

### 4）复述确认 → 直接执行

答齐了，用 1~3 句把需求复述一遍（含关键口径 + 啥算做完），确认后**直接接对应财务技能干**（应收合并 / 按销售拆分 / 劳务发票核对 / 申报表重命名…），不用他再去别处复制粘贴。

## 问法模板（财务场景）

> 「动手前先确认几点（回 `1a 2b`，或回『默认』即可）：
> 1）这活是？　a) 核对发票（劳务核对）　b) 申报表重命名　c) 应收合并　d) 不确定——你帮我判断
> 2）文件在哪？　a) 桌面某文件夹（说下夹名）　b) 我拖进来
> 3）有没有特殊口径要先说？　a) 没有，按常规（默认）　b) 有：____」

## 别犯的错

- ❌ 看一眼文件 / 目录就知道的，还专门问（先看，别问废话）。
- ❌ 该用多选 / 是否题快速消歧的，问成开放式大问题。
- ❌ 信息不全，硬猜一个意图就开干。
