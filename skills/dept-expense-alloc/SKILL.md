# 部门费用归集分摊

把总账会计（斯佳姐）每月初手工做的「损益按部门归集」自动化：**技能包、她自己拖文件跑**，数据只在本机。

**产出**：`部门科目余额表`（左主体 / 中合计 / 右部门 / 费用合计 / 核对）+ `利润表`。  
**验收**：与她的定稿同结构；**核对列 ≈ 0**（容差见 config）；禁止改数凑平。

## 你（agent）该怎么干

**用户是财务同事，不是技术员。** 她丢来一个装着当月材料的文件夹，你认文件 → 跑脚本 → 交出表 + 运行报告。别让她改文件名、看列号。

### 0. 红线
1. **所有金额只由 `scripts/allocate.py` 计算**，你不许心算、估算、手填。
2. **禁止把金额明细、员工姓名打到对话里**；只说 sheet 名、行数、校验通过/差额科目编码。
3. **核对不平禁止改数凑平**；列出差额科目，停下来问她。
4. 真实数据不进 git、不上传外网。

### 1. 盘点输入
材料约定文件夹（有则认，名称近似也行）：

| 夹/角色 | 内容 |
|---------|------|
| 01 主体余额表 | 甲骨易/文化/上海 U8 发生额及余额表 |
| 02 外地代理 | 山东/湖南等代理或出纳表 |
| 03 收入底稿 | 开票 + 微信支付宝 + 无票外汇 |
| 04 人员归属 | 姓名→中心→子部门 |
| 05 用友按人 | 按人辅助核算/明细账 |
| 06 工资社保 | 工资表、社保表 |
| 07 定稿对照 | 可选，用于逐格 diff |

```bash
python3 "<本skill目录>/scripts/inspect_inputs.py" --input-dir <她放文件的绝对目录>
```
- 读 `工作区/output/运行报告_盘点.txt`（或脚本打印）。
- **缺关键输入**（主体余额/人员归属/收入底稿）：用人话列出缺什么，**停**，不要空跑。
- 不歧义则一句话确认："已识别主体×份、归属表、收入底稿，开始归集。"

### 2. 跑归集
```bash
python3 "<本skill目录>/scripts/allocate.py" \
  --input-dir <绝对目录> \
  --out <交付绝对路径>/部门科目余额表_YYYYMM.xlsx
```
- 规则在 `config/业务规则.md` + `表结构模板.json` + `部门名映射.json`。
- **⚠ 5401 成本**：她回填与 9 月定稿冲突——脚本在规则未决前**不对 5401 整挂**；有定稿 diff 时据实报告。
- 5502/5503 按人拆：缺 05/06 时相关科目部门列留空并在报告标明「缺按人输入」。

### 3. 收尾（人话）
- *"本月部门表已出：核对全平 / 有 N 个科目差额（只报编码）。文件在 &lt;路径&gt;。运行报告在同目录。"*
- 有定稿对照时：报「与定稿不一致的科目编码列表」，不报金额。
- 像同事一样：缺文件、未匹配姓名个数、5401 规则未决——主动点出，别闷头当成功。

### 4. 口头改规则 → 只改 config
- 整挂/按人/容差/部门名映射 → `config/业务规则.md`、`部门名映射.json`
- 表头别名 → `config/列名别名.json`
- **改完重跑**，确认后 `git add config && git commit`（无真实数据）。

## 会变 vs 写死
| 写死在脚本 | 外置 config |
|------------|-------------|
| 读余额表位置、合计与核对计算、模板行列骨架 | 科目归集规则、部门映射、列别名、容差 |
| 禁止凑平、缺人停止 | 5401 等待确认项 |

- 真验收：用她 9 月定稿 + 完整输入包，**逐格复现**（材料到齐后做）。
