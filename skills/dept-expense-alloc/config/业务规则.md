## 可调参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 金额容差（元） | 0.02 | \|主体合计 − 部门费用合计\| ≤ 此值视为平 |
| 数据外传 | 禁止 | 只本机；对话不打印金额/人名明细 |

## 交付与范围

| 项 | 口径 | 状态 |
|----|------|------|
| 主产出 | **部门科目余额表** + **利润表**；**所有主体合计 = 所有部门合计**（左=右） | ✅ |
| 工资社保 | **仅在归集流程里必须处理**工资社保数据（**不另出**工资社保报表） | ✅ 标黄澄清 |
| 工资社保数据源 | **也是用友核算数据**（与辅助核算同源账套） | ✅ 标黄澄清 |
| 收入情况 sheet | **不处理** | ✅ |
| 交付 | 技能包，她自己拖文件跑 | ✅ |
| 凑平 | **禁止**改数凑平 | ✅ |

## 表结构

- **左** = 公司主体；**右** = 组织部门  
- **铁律**：所有主体合计 = 所有部门合计  

## 科目 → 部门

| 科目 | 规则 | 状态 |
|------|------|------|
| 5101 主营业务收入 | 开票+微信支付宝+无票外汇 → 营销人 → 人员归属 → 部门 | ✅ |
| **540101、540103**（成本明细） | **整挂「项目总监及助理」** | ✅ 标黄澄清 |
| **5401 其余明细**（工资/社保/差旅等） | **按人员挂部门**（不是整挂项目总监） | ✅ 标黄澄清 |

| 5501 销售费用 | 按人拆到营销子部门 | ✅ |

### 按人拆 · 金额从哪来

- **主源：用友辅助核算**（账套内）  
- 工资/社保相关同样来自用友核算，**归集时必须处理**，不是另做一套工资社保产出  

### 人员 → 部门

- 姓名 → 组织架构1 → 组织架构2；优先架构2  
- 对不上 → 运行报告，禁止猜测  

## 红线

1. 金额仅由 Python 脚本计算  
2. 左≠右 → 列科目编码+差额，禁止凑平  
3. 真实数据不进 git
