# 银行日记账 · 收入提取汇总

出纳的银行日记账是一个 Excel、**每个 sheet 是一个账户**（现金、备用金、工行、中行、美元…）。
每个 sheet 逐笔记流水，用「类型」列标了"收入 / 手续费 / 支出…"。她要的是：**把所有 sheet 里
"收入"那些笔挑出来、汇到一张总表**，供下一步（录入系统建回款）用。这活规则清晰、能高度自动化。

**输出总表列**：`渠道（银行/账户）| 日期 | 到账客户名称 | 金额 | 币种`，按 sheet 在工作簿里的
先后顺序排。

## 你（agent）该怎么干

**用它的是财务出纳。她给一个日记账 Excel，你直接跑出汇总表交付——别让她看命令、看列号。**
大部分笔你直接搞定；只有"我不敢替你定"的少数可疑行，挑出来问她。

### 1. 找输入
用户说文件在哪（"桌面那个日记账""./日记账.xlsx"）就用那个绝对路径。**只需要输入文件，不用她做别的。**

### 2. 跑提取
```bash
# 把 <本skill目录> 换成这份 SKILL.md 所在文件夹的绝对路径
python3 "<本skill目录>/scripts/extract_income.py" <日记账.xlsx 绝对路径>
```
- 输出默认叫 `收入汇总_<输入名>_<日期>.xlsx`，放在输入文件同目录；想指定就加 `-o <路径>`。
- 脚本自动：逐 sheet 找表头→按「类型=收入 且 金额(借方)有数」筛→**跳过"类型=收入但金额空"的
  预填行**（最容易错的地方）→ 客户名取「摘要」、金额取「借方」、渠道取 sheet 名、币种按规则识别
  → 出「收入汇总」+（有可疑行才出）「待人工核对」两个 sheet。

### 3. ⭐ 人在环：只有『待人工核对』里的才问她
脚本会把这两类挑出来（**这些不是错误，是"我不敢替你定"**）：
1. **疑似漏标收入**：某行**有进账金额、却没标成"收入"、也不是手续费/利息等已知支出** → 可能她漏标了。
   这类**不会**被塞进汇总表，只列出来问她："工行第X行有笔 999 元进账没标类型，是不是漏了个收入？"
2. **收入行缺客户名**：标了收入、有金额、但「摘要」空 → 这笔**已进**汇总表（钱是真的），但客户名空，
   下一步去系统建回款要用到名字，所以标黄+列出来提醒她补。
- **别问的**：识别规则本身、汇总表怎么出、每笔叫啥——跑完一句话汇总即可。

### 4. 像同事一样收尾（人话，主动但克制）
- 全部干净、没待人工 → 一句话："**收入提取好了，N 笔（人民币 X 笔、外币 Y 笔），在 `收入汇总_…xlsx`。**"
- 有待人工 → 才提，且只报那几条+原因，别把整张表念一遍。
- **该主动操心的**：① 多币种时点一句"美元没跟人民币混加，分开列了"；② 某个 sheet 被整个跳过
  （没识别到表头/缺列）一定要说，别让她以为那个账户没收入其实是没读到；③ 某 sheet 收入笔数明显
  异常（比如平时几十笔这次 0 笔）可以问一句。

## 会变的东西在哪（改表不改码）
全在 `config/识别规则.md`，改完重跑即可，也可以**直接对我口头说**（"把'退汇'也算收入""客户名改从'对方户名'列取"）我来改：
- **收入判定关键词**（默认「收入 / 到账」）——她用别的词标收入就加。
- **支出类关键词**（手续费/利息/支出…）——只影响"漏标"检测，加得越全、误报越少。
- **列名识别词**：日期列 / 客户名列 / 金额列 / 类型列 各认哪些表头名——换了格式改这里。
- **跳过的 sheet**：表名含"说明/目录/汇总表…"的不当日记账。
- **外币 sheet**：`关键词 => 币种`，认外币账户。

> 列是**按表头名字找**的、不认死列位置，所以列顺序有出入也能认；找不到表头或缺关键列的 sheet
> 会被安全跳过并在小结里说明——不会静默丢数据。

## 验收口径
`tests/test_extract.py`（合成数据，14 项断言）覆盖：空预填行跳过、多笔、缺日期保留、手续费排除、
多币种不混加、缺客户名+疑似漏标两类 flag、说明页跳过。跑出 `All passed ✅` 即回归通过。
换真实批次：看脚本报的收入笔数/各 sheet 小结、抽一两笔对一下日记账即可信。
