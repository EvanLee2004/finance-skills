# 收入提取 · 识别规则（活表）

> 这张表控制"怎么认收入、怎么认列、哪些 sheet 跳过、哪些 sheet 算外币"。
> **改表不改码**：日记账换了格式（列名不一样、多了新银行、用别的词标收入）时，
> 改这张表、重跑即可，不用动脚本。也可以直接对 opencode 口头说"把 XX 也算收入"，
> 让它帮你改这里。
>
> 格式：每节下面**一行一个词**（`- ` 开头）。外币那节用 `关键词 => 币种`。

## 收入判定关键词
> 「类型」列的值里**含**下面任一词，就算这行是收入。
- 收入
- 到账

## 支出类关键词
> 已知**不是**收入的类型。用途：做"漏标"检测时，有金额但类型是这些的，不当成疑似漏标。
- 手续费
- 利息
- 支出
- 转出
- 服务费
- 退款
- 费用

## 日期列表头词
> 表头叫这些名字之一，就认成"日期"列。
- 日期
- 交易日期
- 记账日期

## 客户名列表头词
> 认成"到账客户名称"的来源列。
- 摘要
- 客户
- 对方户名
- 汇款人
- 收款人
- 对方

## 金额列表头词
> 认成"收入金额"的来源列（收入记在借方/增加方）。
- 借方
- 增加
- 收入金额
- 收方

## 类型列表头词
- 类型
- 类别

## 跳过的 sheet
> 表名**含**这些词，就不当日记账处理（如封面、说明页、月度汇总页）。
- 说明
- 目录
- 封面
- 模板
- 汇总表

## 外币 sheet
> 表名或行内标记**含**左边关键词的 sheet，整表标为右边的币种。
> 没命中任何一条的 sheet 默认「人民币」。
- 美元 => 美元
- 美金 => 美元
- USD => 美元
- 欧元 => 欧元
- EUR => 欧元
- 英镑 => 英镑
- GBP => 英镑
- 港币 => 港币
- 港元 => 港币
- HKD => 港币
- 日元 => 日元
- JPY => 日元
