# 代扣代缴申报表批量重命名

把财务部手工给「代扣代缴、代收代缴税款报告表」PDF 一个个改名自动化。这批 PDF 由税务系统（自然人电子税务局/扣缴端）导出，**结构固定**：2 页，第1页汇总、第2页明细。每份对应一笔对外付款的代扣代缴记录。

**重命名规则**：`新名 = {被代扣代缴代收代缴纳税人名称}{计税依据合计 + 实代扣代缴税额合计}.pdf`
- 名称 → PDF **第2页**表格列2（英文公司名/人名，单元格里被换行切碎，要合并+按业务词表还原空格）
- 金额 → PDF **第1页**合计行（含「合/总」字的行）列5（计税依据）+ 列10（实缴税额），相加、去尾零

例：`1,415.47 + 84.93 = 1500.4` → `LUCALIZE MANAGEMENT CONSULTANCIES CO.L.L.C1500.4.pdf`

## 你（agent）该怎么干

**核心原则：用它的是财务同事。她给一个装着一堆申报表 PDF 的文件夹，你改好名、产出对照表交付——别让她看命令、看表格列号。** 这活规则清晰、能高度自动化，**大多数文件你直接搞定**；只有少数抽不出名/金额的，别瞎命名，挑出来问她。

### ⭐ 人在环：默认不动原件，疑难才问
- **自己干**：找输入文件夹 → 逐个 PDF 抽名+抽金额 → 算新名 → **复制**改名（原件不动）→ 出对照表。
- **默认走 copy 模式**（脚本默认值）：改好名的副本进新文件夹 `重命名_<日期>/`，**原 PDF 原封不动**，万一哪个名不对，原件还在、对照表能回溯。**别默认就地改名**——除非用户明确说「就地改 / 直接重命名原文件夹」，那才加 `--mode rename`（脚本会先自动整批备份再改）。
- **只在这几种情况问她**：
  1. **有「待人工确认」的文件**（脚本会单独列出：没抽到名、没抽到合计金额、或名称里带 `?` 等可疑）→ 这些**没改名**，问她："有 N 个没认出来（XX 抽不到名/金额），你看下这几个的正确名？"
  2. **冒出没见过的公司后缀词**导致某个名没正确分词（几个词黏在一起没空格）→ 把那个词补进 `config/名称分词词表.md` 重跑，或问她确认。
- **别问的**：改名规则本身、对照表怎么出、各文件叫啥——跑完一句话汇总即可。

### 1. 找输入（装 PDF 的文件夹）
**用户说文件夹在哪（"在桌面那个申报表文件夹""在 ./申报表/"），就直接用那个绝对路径**传 `--input`。整个文件夹里的 `.pdf` 都会被处理。

### 2. 跑重命名
```bash
# 把 <本skill目录> 换成这份 SKILL.md 所在文件夹的绝对路径
python3 "<本skill目录>/scripts/rename.py" --input <PDF文件夹绝对路径>
```
- **默认 copy**：副本写进 `<input同级>/重命名_<日期>/`，原件不动。想指定交付夹就加 `--out-dir <干净目录>`。
- **想先看一眼不落地**：加 `--dry-run`，只打印「原名 → 新名」和待人工清单，不动任何文件。**第一次对新一批文件，先 dry-run 看一眼更稳。**
- **用户要就地改名**：加 `--mode rename`（脚本先把原件整批复制到 `申报表_备份_<日期>/`，再在原文件夹里 rename）。
- 脚本自动：抽名（合并换行+按 `config/名称分词词表.md` 分词）→ 抽第1页合计金额 → 相加去尾零 → 重名加 `_1/_2` → 出 `对照表.csv`（含待人工标注）。

### 3. 跑完收尾（人话，给摘要）
看脚本输出的「可重命名 N / 待人工 M」：
- **全部 ✓、没待人工** → 一句话带过："重命名好了，N 个，在 `重命名_<日期>/`，对照表也在里面，原件没动。"
- **有待人工** → 才提她（见人在环），把那几个文件名+原因报给她，别一股脑甩全清单。
- 别把 N 个文件的新名逐条念给她，她要的是"改好了、几个、在哪、有没有没认出来的"。

### 3.5 ⭐ 像同事一样收尾（数字员工三件事 · 主动但克制）
1. **环比/异常小结**：若知道上次这批多少份，提一句"这次 **N 份**（上次 M）"；某份金额异常大/为 0 也可点一句。
2. **拿不准就提具体问题**：某个名抽出来是空/带 `?`/几个词黏一起——别默默生成乱名，挑出来问"这个 PDF 的纳税人名我没认准，是不是 XX？"
3. **本岗位该主动操心啥**：**合计行没认出来**（金额抽不到 → 是不是这份 PDF 结构不一样）、**整页纳税人名缺失**、**同名冲突**（脚本加了 `_1` 后缀，提醒她核一下是不是真两笔）。

## 会变的东西在哪（改表不改码）
- **公司后缀词表**（用来给黏在一起的英文名还原空格）→ `config/名称分词词表.md`（一行一个全大写词，加词重跑即可）。
- **疑难名手工映射**（算法搞不定、又不想每次进待人工的）→ `config/名称修正表.md`（原文件名 → 正确新名，初始为空）。
