# withholding-report-rename · 代扣代缴申报表重命名

把一批「代扣代缴、代收代缴税款报告表」PDF 批量重命名成
`{纳税人名称}{计税依据合计+实缴税额合计}.pdf`。

- **干什么**：名称取自 PDF 第2页明细、金额取自第1页合计行；自动合并被换行切碎的英文名、相加金额、去尾零。
- **默认安全**：copy 模式——改好名的副本进新文件夹、原件不动，并产出 `对照表.csv`。
- **怎么跑**：见 `SKILL.md`。核心 `python3 scripts/rename.py --input <PDF文件夹> [--dry-run] [--mode copy|rename]`。

## 术语
- **代扣代缴、代收代缴税款报告表**：税务系统导出的 PDF，2 页，第1页汇总、第2页明细，每份=一笔对外付款的代扣代缴申报。
- **计税依据合计 / 实缴税额合计**：第1页合计行的列5、列10，二者相加即文件名里的金额。
- **被代扣代缴代收代缴纳税人名称**：第2页列2的境外公司/个人名。

## 文件
- `scripts/rename.py` — 核心脚本（两阶段：先读全部算新名、再统一 copy/rename，避免文件锁）。
- `config/名称分词词表.md` — 活表：公司后缀词（还原英文名空格）。
- `config/名称修正表.md` — 活表：疑难名手工映射（初始空）。
- `tests/test_rename.py` — 回归：5 个样本 PDF → 5 个已知新名。

依赖：`pdfplumber`（见 env-doctor 依赖清单）。
