# SOP · 追觅应收进度多版本对比

> 给 agent / 维护者看的完整口径。**日常跑活只读 SKILL.md**（已含冷启动 briefing）；改逻辑、排错、验收时读本文。  
> 本文里的「金标数字」只用于回归，**不是业务常量**。

## 1. 业务背景

财务同事（亮晶）维护追觅客户的「应收进度」宽表：行=账期（去年×月、1月…），列=各业务线/对接人（金额 + 常挂 PO时间 / 发票上传时间 / 预计付款时间）。  
不定期导出一版 list（文件名**可能**含日期，但**不要写死某套文件名**）。需要知道相对更早快照发生了什么：催促、回款、收 PO、底色状态、新月补录、列增删改名。

手工逐格比费时且易漏（尤其新月、改名列、公式格）。本技能 = 稳定 diff 流水线 + agent 人话解读。

## 2. 源表结构（约定）

- 工作簿内目标 sheet 名默认 **「应收进度」**。
- **第 2 行**：人名/业务列标题；其右可跟子列标题（PO时间 等）。
- **第 3 行起 A 列**：期间标签，直到碰到停止词（「为方便…」「总计」等）为止。
- 末尾可能有**无子列的单列**（仅金额格）。
- 单元格可有**填充色**表示状态（绿待付 / 黄未收PO / 橙流程中 / 灰已回款等）。
- 「预计付款时间」常写 **Excel 公式** `=发票上传时间+60`，业务上不算「备注变更」。

## 3. 处理步骤（脚本 S1–S7）

1. **S1 加载配置**：`业务规则.md` / `子列识别.json` / `颜色图例.json`。  
2. **S2 打开文件**：只读；定位 sheet；解析列组与期间。  
3. **S3 标签**：从文件名抽 MMDD 等标签；重名加后缀。  
4. **S4 对齐**：以最早版本为锚，人名归一化 + 核心名匹配；一对一；独有列单独挂。  
5. **S5 期间并集**：按锚文件顺序，后文件新期间追加（**禁止只取交集**）。  
6. **S6 Diff**：  
   - 值：`norm_compare`（公式→空，日期→YYYY-MM-DD，数稳格式化）  
   - 色：同时存在期间的版本间 bg RGB  
   - 付款备注：仅非公式文本  
7. **S7 出报告**：多 sheet + 结论摘要；源文件不动。
