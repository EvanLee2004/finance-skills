## 一、可调参数（脚本读取生效）

| 参数 | 值 | 说明 |
|------|----|------|
| 目标 sheet 名 | 应收进度 | 优先精确匹配；找不到再模糊包含「应收进度」的 sheet |
| 表头所在行 | 2 | 人名列 / 子列标题所在行（1-based） |
| 数据起始行 | 3 | 期间列（A 列）数据从第几行开始 |
| 期间列 | 1 | 期间标签所在列（通常 A=1） |
| 人名起始列 | 2 | 第一个人名/业务列（通常 B=2） |
| 子列最大个数 | 3 | 金额列右侧最多认几个子列（PO/发票/预计付款） |
| 忽略公式 | 是 | 对比文本时：公式（以 `=` 开头）视为空，只比备注文本 |
| 明细只含有变化或有文本 | 是 | 明细对比不灌空灰底格，只保留有变化或有非空文本的行 |
| 输出含文字结论 sheet | 是 | 在 Excel 报告最前增加「结论摘要」sheet |

> 🔧 改参数：直接改上表「值」列。脚本读不到就用内置默认。

## 四、人名对齐规则（写死算法，这里解释）

跨文件对齐人名列时：

1. **完全相同**（归一化后：小写、去空格/换行、全角括号→半角）→ 满分匹配  
2. **去括号核心名相同**（如 `MOVA扫地机(朱海心)` 与 `MOVA扫地机(朱海心，中华)`）→ 高分匹配  
3. **一方包含另一方** → 中分匹配  
4. 已匹配的列不再给别人用（一对一）  
5. 只在某版本出现的列单独列出「缺失/新增」

拿不准（两个候选同分）→ 运行报告标「对齐存疑」，agent 问人。

## 五、对比口径（确定性，写死在脚本）

| 维度 | 口径 |
|------|------|
| 期间 | 所有输入文件的期间**并集**（新月不会漏） |
| 值变化 | 非公式文本/数字/日期规范化后不一致，且至少一方非空 |
| 底色变化 | 在「同时存在该期间」的版本之间，背景色 RGB 集合 > 1 |
| 预计付款备注 | 忽略公式后的文本 diff；纯公式有无差异不计入备注变化 |
| 新增期间 | 某版本无该期间、其他版本有非空内容 → 记为值变化（如 6 月新账） |

---

## 六、范围与边界

- 只对比「应收进度」类宽表（人名列组 + 期间行）；其他 sheet（业务明细、账单汇总）**不碰**。  
- 默认 2～N 个版本（常见 3 个：旧/中/新）；**至少 2 个**。  
- 文件命名可含日期（`0703`/`20260703`/`7.3`），脚本尽量抽标签；抽不到用 `v1/v2/…`。  
- 不修改源文件（只读）；产物只写 `--out` 或源目录/工作区 output。
