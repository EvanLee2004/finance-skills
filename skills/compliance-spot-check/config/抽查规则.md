## 一、可调阈值（脚本读取生效 · 与〇不重复）

| 参数 | 值 | 说明 |
|------|----|------|
| 账龄门槛月数 | 6 | 普通客户账龄 ≥ 此值 → 账龄达标 |
| 金额绝对兜底 | 10000 | 该客户该月合计 ≥ 此值 → 金额达标（与账龄 OR） |
| 销售内相对金额门槛 | 0.5 | 金额分位 ≥ 此值（0~1）→ 相对金额达标（小盘销售用） |
| 相对金额最低账龄 | 3 | 走「相对金额达标」时，账龄至少也要到此月数（避免全是 0~1 月小单的销售被相对分位硬抬进名单） |
| 每人最少建议条数 | 1 | 有资格候选时，每位在职销售至少进名单条数 |
| 未反馈加成 | 1.5 | 该销售存在「未反馈」历史时，其名下候选分数 × 此倍数 |
| 已抽过处理 | 排除 | 已抽过且非「未反馈」的 (销售,客户,月份) 不再进主名单 |
| 忽略销售桶 | 高美杰1 | 坏账桶等整行忽略（与 split-by-sales 一致） |

> 🔧 改参数：直接改上表「值」。读不到则用内置默认。

## 二、特殊客户怎么算账龄（解释；名单见〇）

> 〇 里名单中的客户：**不用绝对 6 个月卡死**，改用「该客户样本账龄中位数（至少 1）」——账龄 ≥ 中位才算账龄达标。

## 五、抽查历史文件格式（技能自建）

CSV（UTF-8，可带 BOM），列：

| 列 | 说明 |
|----|------|
| 销售 | 营销人员名 |
| 客户 | 客户名称 |
| 交付月份 | YYYYMM 如 202604 |
| 抽查日期 | 可选，YYYY-MM-DD |
| 反馈状态 | `未反馈` / `已反馈`（空当已反馈，不再优先） |

首次可无文件；技能照常跑。

---

## 六、资格规则（写死逻辑 · 阈值见上表）

单位 = 销售 + 客户 + 交付月份（金额合计、账龄取最大）。

**够格（能进候选）满足任一即可：**

1. 账龄达标（普通 ≥ 账龄门槛；特殊客户 ≥ 该客户中位数）  
2. 金额 ≥ 绝对兜底  
3. 销售内金额分位 ≥ 相对门槛 **且** 账龄 ≥ 相对金额最低账龄  

然后：未反馈优先加成、离职/忽略跳过、已反馈排除；有资格者每人≥1 再按分填满每周条数。  
输出：营销人员｜客户名称｜交付月份（+ 可选理由），月份 `YYYYMM`。
