## 一、可调阈值（脚本读取生效）

| 参数 | 值 | 说明 |
|------|----|------|
| 账龄门槛月数 | 6 | 普通客户账龄 ≥ 此值 → 账龄高分（录音原话 6 个月） |
| 金额绝对兜底 | 10000 | 该客户该月合计 ≥ 此值 → 金额直接高分，即使账龄不长也可抽（OR 关系） |
| 每周建议条数上限 | 20 | 同「每周抽查条数」；覆盖每人≥1 后按分填满 |
| 每人最少建议条数 | 1 | 覆盖约束：每位在职销售（有候选时）至少进名单条数 |
| 未反馈加成 | 1.5 | 该销售存在「未反馈」历史时，其名下候选分数 × 此倍数 |
| 已抽过处理 | 排除 | 已抽过且非「未反馈」的 (销售,客户,月份) 不再进主名单 |
| 忽略销售桶 | 高美杰1 | 坏账桶等整行忽略（与 split-by-sales 一致） |

> 🔧 改参数：直接改上表「值」。读不到则用脚本内置默认（6 / 10000 / 20 / 1 / 1.5 / 排除 / 高美杰1）。

## 五、抽查历史文件格式（技能自建，见「历史表衔接方式」默认）

CSV（UTF-8，可带 BOM），列：

| 列 | 说明 |
|----|------|
| 销售 | 营销人员名 |
| 客户 | 客户名称 |
| 交付月份 | YYYYMM 数值如 202604 |
| 抽查日期 | 可选，YYYY-MM-DD |
| 反馈状态 | `未反馈` / `已反馈`（空当已反馈处理，不再优先） |

首次可无文件；技能照常跑，仅无「未反馈优先 / 已抽过排除」信息。

## 六、确定性规则摘要（写死在脚本，此处便于核对）

1. **抽查单位** = 销售 + 客户 + 交付月份（同月同客户多单金额合计、账龄取最大）。
2. **金额分** = 该销售内部相对分位 + 绝对兜底（≥1 万高分）。
3. **账龄分** = 普通客户 ≥6 月高分；特殊客户相对化。
4. **资格**（进候选）：账龄达标 **或** 金额≥兜底 **或**（特殊客户且相对账龄达标）；否则低分仅在覆盖不足时兜底。
5. **未反馈优先**、**离职跳过**、**覆盖全部在职（有候选者）每人≥1 再按分填满**。
6. 输出文字列：**营销人员｜客户名称｜交付月份**（+ 可选理由列），交付月份 `YYYYMM`。
