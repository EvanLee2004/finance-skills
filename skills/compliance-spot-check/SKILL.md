# 合规文件抽查（本周建议清单）

每周财务要从应收订单里挑一批，通知对应销售提供合规文件（合同/对账记录），核查「合同与订单是否匹配」。本技能 = **推荐工具**：算出一份「本周建议抽查名单」，**人工再从建议里挑一挑、发邮件、人工核合同**——不自动发邮件、不自动判匹配。

## 你（agent）该怎么干

**核心原则：用它的是财务同事，不是技术员。** 她给一张应收 all（可选再给抽查历史），你产出文字版建议清单——别让她看命令、看 sheet 名、自己算谁该抽。

### ⭐ 人在环：什么自己干、什么问她

> 原则：**确定的、可逆的、常规的 → 自己干**；**只有她才知道的、最终拍板 → 问她**。本技能产出的是**建议**，最终抽谁由她定。

**自己干**
- 认应收 all、读配置、按规则打分覆盖、写清单、报摘要。
- 历史文件不存在 → **按无历史继续跑**（别拦她）。
- 命令、列名、阈值 —— 你消化。

**问她（克制）**
1. 跑之前若她没提：轻问一句有没有本周要优先盯的销售/客户；没有就默认规则跑。
2. **某销售一个候选都没有**（日志会提示）→ 用人话问："测销戊名下都是短账龄小额，这周先不抽他可以吗？还是把他也算特殊？"
3. **冒出陌生「方圆型」短账龄大客户** → 问要不要加进 `config/抽查规则.md` 特殊客户名单。
4. 清单出来后：**让她从建议里挑**，别替她做最终决定。她口头改条数/特殊客户/离职名单 → 你改 config → 重跑。

### 1. 找输入
需要：①**应收 all**（`receivables-merge` 产物，含销售/客户/交付月份/账龄/金额）②（可选）**抽查历史** CSV（技能自建格式，见 config）。

⚠️ **路径全用绝对路径。** 脚本/配置在**「这份 SKILL.md 所在文件夹」**，`<本skill目录>` 换成真实绝对路径。

```bash
python3 "<本skill目录>/scripts/recommend.py" \
  --input <应收all绝对路径> \
  --history <抽查历史csv绝对路径，可省> \
  --out <交付目录绝对路径>/本周抽查建议.txt
```

- 用户说文件在哪就直接用；没说才去 `工作区/input/` 取最新 xlsx。
- `--history` 可省；给了但文件不存在 → 脚本警告后按无历史继续。
- `--out` 可省：默认落在 all **同目录** 的 `本周抽查建议_<文件名>.txt`；也可 `.xlsx`（会多写同名 `.txt` 方便粘贴）。

### 2. 脚本自动做的事
1. 认数据 sheet、映射列（与应收技能同一套 17 列口径；必填：销售人员/客户名称/应收金额/交付月份/账龄）。
2. **按 销售+客户+交付月份 聚合**（同月同客户多单金额合计、账龄取最大）。
3. 读 `config/抽查规则.md`：离职跳过、特殊客户账龄相对化、金额相对分+1万兜底、未反馈加成、覆盖在职每人≥1 再填满每周上限。
4. 输出文字列：**营销人员｜客户名称｜交付月份**（默认再加**理由**列，纯文字；可在 config 关掉）。

### 3. 跑完收尾（人话）
- *"本周建议 N 条，覆盖 M 位销售，在 <路径>。未反馈优先带了 X；Y 名下没有符合条件的单。你从里面挑要通知的就行。"*
- 把清单正文给她（或路径）；**别替她发邮件**。
- 她确认「这几条真抽了」→ 帮她把结果回填进抽查历史 CSV（销售|客户|月份|日期|反馈状态=未反馈），下周才会未反馈优先/已抽排除。

### 3.5 ⭐ 像同事一样收尾（数字员工三件事）
1. **环比小结**：若有上周清单/历史，提一句新覆盖了谁、谁连续未反馈。
2. **拿不准提具体问题**：又一个短账龄大客、或某销售全员金额极小排不出候选。
3. **该操心啥**：连续多周未反馈、金额很大却从未抽过、离职名单漏跳过。

## 会变的东西在哪（改表不改码）
- **阈值/覆盖/未反馈加成/离职/特殊客户/每周条数/是否带理由/验收人** → `config/抽查规则.md`（含方案 5 个待确认默认值，旁注「待确认·默认取X」）
- **列名别名** → `config/列名别名.json`
- 确定性算法（聚合、相对金额分、覆盖填充）在 `scripts/recommend.py`。

## 验收口径
合成数据（`技能/合规文件抽查/测试数据/`，虚构名）：
- 离职销售 / 坏账桶不出现在清单。
- 小盘销售（金额全 <1万）仍可因**销售内相对金额**进名单。
- 特殊客户短账龄不被 6 个月卡死。
- 历史「未反馈」销售优先；已反馈的 (销售,客户,月) 排除。
- 历史文件缺失不崩；缺必要列清晰报错。
- `python3 tests/test_robustness.py` 全绿。

> ⚠ 真实应收数据勿进仓库；本技能建议清单仅供内部抽查，勿外传客户明细。
