# labor-invoice-check（劳务发票核对）

个人劳务付款前的发票核对（第一阶段）。给同事用 opencode 说人话跑；也可命令行直接跑。

## 干嘛的
两张表 → 一张核对结果：
- 输入：**待支付清单**（国内个人：姓名/应付/备注/开户名/证件号…）+ **发票统计台账**（销售方名称/纳税人识别号/合计金额）。
- 处理：按**身份证号**求和多张发票（未命中**姓名兜底**）；实习生/外国人豁免、公司·对公待人工、≤800放行、>800缺票/未开票标黄。
- 输出：`主核对表` + `不付名单(催票)` + `可付名单` + `运行报告`（含开户名、匹配方式）。

## v1.1（2026-07 亮晶真实验收补洞）
| 坑 | 修法 |
|----|------|
| 中文名外籍被催票 | 外国人三信号：供应商英文(去括号昵称)/开户名英文/护照号 |
| 张佳音有票判无票 | 身份证主匹配 + 姓名兜底（个体户税号） |
| 公司行混进催票 | 公司关键字 / 支付渠道=对公 → 标黄-待人工 |
| 看不出代收 | 开户名≠供应商且双中文 → 提示「疑代收」 |

## 命令行
```bash
python3 scripts/check.py --inspect --input-dir <目录>
python3 scripts/check.py --list 待支付清单.xlsx --invoice 发票台账.xlsx --out 结果.xlsx
```

## 跑测试
```bash
python3 tests/test_robustness.py   # 合成金标 + 真实冒烟 + 本月亮晶金标（若 Downloads 有文件）
```

## 改规则（不改代码）
- 门槛/容差/实习生/公司关键字 → `config/业务规则.md`
- 列名认别名 → `config/列名别名.json`

## 结构
```
labor-invoice-check/
├── SKILL.md            给 agent 的操作说明
├── scripts/check.py    核对核心（确定性、可复现）
├── config/             业务规则.md + 列名别名.json
├── tests/              test_robustness.py

```
