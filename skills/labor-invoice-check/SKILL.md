# 劳务发票核对（个人 · 支付前）

把财务部每月给个人供应商（freelancer/实习生）付译费/劳务费前手工核对开票的活自动化。
背景：大于 800 元的个人对外提供劳务依规需开增值税发票；付款前要确认谁足额开票了，没开/不够的标黄、暂不付、催票。

## 你（agent）该怎么干

**核心原则：用它的是财务同事（赵亮晶），不是技术员。** 她扔两个 Excel 进来，你把活办完——认文件/跑脚本/出复核全归你；别让她改文件名、看命令、看 sheet 名、看身份证号。

### 1. 找齐两个输入（按内容认，不靠文件名）
需要：①**待支付清单**（本月要付的「国内个人」表：供应商姓名/应付金额/备注/身份证号…）②**发票统计台账**（资源部出的：销售方信息名称/纳税人识别号/合计金额…）。

⚠️ **路径全用绝对路径，别 cd、别硬编码别人机器的路径。** 脚本/配置就在**「这份 SKILL.md 所在文件夹」**里，脚本按自身位置自动找 `config/`。下面 `<本skill目录>` 换成真实绝对路径。

```bash
python3 "<本skill目录>/scripts/check.py" --inspect --input-dir <放文件的绝对目录>
```
- 默认找 `工作区/input/`；文件在别处就把 `--input-dir` 指过去（绝对路径）。
- 认出来后**不歧义就一句话告知**（"清单=X、发票台账=Y，我按这个核对了"）；认不准/缺文件才让她确认。

### 2. 跑核对
```bash
python3 "<本skill目录>/scripts/check.py" \
  --list <清单绝对路径> --invoice <发票台账绝对路径> \
  --out <交付目录绝对路径>/劳务发票核对结果_<月份>.xlsx
```
- `--list/--invoice` 省略则按内容从 `--input-dir` 自动认。
- `--out` 可省：省了落在**清单同目录**；想放别处用绝对路径。跑完把最终路径明确告诉她。
- 脚本自动：读清单（自动定位表头行、剔除"合计"汇总行）+ 读台账 → **按身份证号把多张发票合计金额求和** → 分类判状态 → 出 4 个 sheet。
- **多 sheet 抗干扰**：发票文件里常带别的 sheet（手工底稿"财务核对"、Sheet2/3 等）——脚本**按列特征**认台账（只认同时含『纳税人识别号』+『合计金额』那张），**无视其它 sheet、也不依赖它叫不叫 Sheet1**；清单侧同理（认含『供应商姓名』+『应付金额』的 sheet）。

### 3. 核对逻辑（脚本自动做，门槛/容差/规则都在 `config/业务规则.md`）
- **匹配键 = 身份证号**（清单`身份证号/护照号` ↔ 台账`销售方信息纳税人识别号`）。比姓名可靠：治重名、姓名特殊字符（"流畅（阿拉伯语）"）、两表写法不一。
- **实习生**（备注含 Intern）→ 豁免不标记；**外国人**（纯英文名）→ 豁免。
- **应付 ≤ 800** → 可付（不要求开票）；**应付 > 800**：开票总额 ≥ 应付−容差 → 可付（含多开，多开留第二轮）；0<开票<应付−容差 → **标黄·缺票**；无票/开票为0 → **标黄·未开票**。

### 4. 出活后怎么收尾（人话、给摘要、别甩原始数据）
跑完一句话报结论 + 重点摆"不付名单"：
- *"这月 464 人核完了：可付 X，标黄 Y（缺票 a + 未开票 b）——不付名单这 Y 个人我列在『不付名单』shet 了，建议交资源部/Mike 催票。结果在 <路径>。"*
- **标黄里有"无身份证号(疑外籍)"提示的**（如中文名实为越南籍、没身份证匹不上），单独提一句让她看护照核实——这是人工兜底，脚本识别不了的就靠她。
- **开票为0(串票/录入)** 的也单独点一句。

### 5. 口头改规则 → 改 config（活表，别碰代码）
她说"门槛改成1000""容差放到5毛""这个备注也算实习生"——你去改 `config/业务规则.md` 里的参数表 → 重跑 → 给她看 → `git add config/ && git commit` 固化。**绝不把规则写死进脚本。**

## 会变的东西在哪（改表不改码）
- **门槛/容差/实习生关键字/外国人识别** → `config/业务规则.md`（可调参数表）
- **两张表的列名认别名** → `config/列名别名.json`（表头变了加别名）
- 确定性算法（按身份证求和、分类判断、剔除合计行）写死在 `scripts/check.py` 保证可复现。

## 验收口径（判断脚本对不对）
**已知基准**（测试数据 `待支付译费.xlsx`(202603) + `个人发票统计.xlsx`）：
- 清单 **464 人**（已剔除表底"合计"行）；**可付 229、标黄-未开票 16、标黄-缺票 2、豁免-实习生 210、豁免-外国人 7**；标黄合计 **18**。
- 关键案例：**杨仲舒**应付3682.36、两张票2091.52+1590.84求和=3682.36 → **可付**（验证一对多求和）；**费祥汝**1148无票 → **未开票标黄**；各 Intern → 不标记。
- 跑 `tests/test_robustness.py` 应全绿（合成金标 + 真实冒烟）。

换别月数据跑：看认列无告警、关键案例对、标黄都是真没票/缺票的人，即结构稳。

> ⚠ 数据敏感：清单含身份证/账号/电话 PII，**只在本地跑、结果别外传**；身份证号仅作匹配键，输出默认不展示付款列。
