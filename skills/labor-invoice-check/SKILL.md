# 劳务发票核对（个人 · 支付前）

把财务部每月给个人供应商（freelancer/实习生）付译费/劳务费前手工核对开票的活自动化。
背景：大于 800 元的个人对外提供劳务依规需开增值税发票；付款前要确认谁足额开票了，没开/不够的标黄、暂不付、催票。

## 你（agent）该怎么干

**核心原则：用它的是财务同事（赵亮晶），不是技术员。** 她扔两个 Excel 进来，你把活办完——认文件/跑脚本/出复核全归你；别让她改文件名、看命令、看 sheet 名、看身份证号。

### 1. 找齐两个输入（按内容认，不靠文件名）
需要：①**待支付清单**（本月要付的「国内个人」表：供应商姓名/应付金额/备注/开户名/身份证号…）②**发票统计台账**（资源部出的：销售方信息名称/纳税人识别号/合计金额…）。

⚠️ **路径全用绝对路径，别 cd、别硬编码别人机器的路径。** 脚本/配置就在**「这份 SKILL.md 所在文件夹」**里，脚本按自身位置自动找 `config/`。下面 `<本skill目录>` 换成真实绝对路径。

```bash
python3 "<本skill目录>/scripts/check.py" --inspect --input-dir <放文件的绝对目录>
```
- 默认找 `工作区/input/`；文件在别处就把 `--input-dir` 指过去（绝对路径）。
- 认出来后**不歧义就一句话告知**（"清单=X、发票台账=Y，我按这个核对了"）；认不准/缺文件才让她确认。

### 2. 跑核对
```bash
python3 "<本skill目录>/scripts/check.py" \
  --list <清单绝对路径> --invoice <发票台账绝对路径> \
  --out <交付目录绝对路径>/劳务发票核对结果_<月份>.xlsx
```
- `--list/--invoice` 省略则按内容从 `--input-dir` 自动认。
- `--out` 可省：省了落在**清单同目录**；想放别处用绝对路径。跑完把最终路径明确告诉她。
- 脚本自动：读清单（自动定位表头行、剔除"合计"汇总行）+ 读台账 → **按身份证号把多张发票合计金额求和（未命中再姓名兜底）** → 分类判状态 → 出 4 个 sheet。
- **多 sheet 抗干扰**：发票文件里常带别的 sheet（手工底稿"财务核对"、Sheet2/3 等）——脚本**按列特征**认台账（只认同时含『纳税人识别号』+『合计金额』那张），**无视其它 sheet**；清单侧同理。

### 3. 核对逻辑（脚本自动做，门槛/容差/规则都在 `config/业务规则.md`）
- **匹配** = 身份证号为主 + 姓名兜底（个体户税号≠身份证时，如张佳音）。
- **实习生**（备注含 Intern）→ 豁免；**外国人三信号**（供应商英文名去括号昵称 / 开户名英文 / 护照号）→ 豁免。
- **公司名**或**支付渠道=对公** → `标黄-待人工`（勿当个人催票）。
- **应付 ≤ 800** → 可付；**应付 > 800**：足额可付（含多开）；缺票/未开票标黄。
- 输出列含 **开户名、匹配方式**，方便她一眼复核。

### 4. 出活后怎么收尾（人话、给摘要、别甩原始数据）
跑完一句话报结论 + 重点摆"不付名单"：
- *"这月 N 人核完了：可付 X，标黄-未开票 Y，待人工 Z——不付名单里未开票交资源部/Mike 催票；待人工（对公/公司）你自己定走法。结果在 <路径>。"*
- **主动点名**（脚本 stderr 也会打）：
  - 大额未开票（≥5000，优先催）
  - 姓名兜底命中（疑个体户税号，已按名放行若足额）
  - 待人工名单
- **别把「豁免-外国人」当没开票催**；代收款人中文开户名不等于要票的中国 freelancer。

### 4.5 ⭐ 像同事一样收尾（数字员工三件事 · 主动但克制）
1. **异常小结**：金额特别大的未开票、姓名兜底、待人工、临界 800 附近。
2. **拿不准就提具体问题**：例 *"朱远辰支付渠道写了对公，是不是走公司票/对公支付、不算个人没开票？"*
3. **本岗位留心眼**：连续多月未开票老面孔、开票为0串票、疑外籍但三信号都没沾上的中文名。

### 5. 口头改规则 → 改 config（活表，别碰代码）
她说"门槛改成1000""容差放到5毛""这个备注也算实习生"——你去改 `config/业务规则.md` → 重跑 → 给她看 → `git add config/ && git commit` 固化。**绝不把规则写死进脚本。**

## 会变的东西在哪（改表不改码）
- **门槛/容差/实习生关键字/公司关键字** → `config/业务规则.md`
- **两张表的列名认别名** → `config/列名别名.json`
- 确定性算法（身份证求和、外籍三信号、姓名兜底、对公启发式）写死在 `scripts/check.py`。
