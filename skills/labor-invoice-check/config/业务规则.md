# 劳务发票核对 · 业务规则（活 · 人维护）

> 这是一张「活」表。**会变/要判断的业务参数放这里，脚本读它执行**；规则变 = 改这张表、不改代码。
> 确定性算法（按身份证号求和、外籍三信号、姓名兜底、分类判断）写死在脚本里保证可复现；这里的参数脚本会读取生效。

## 一、可调参数（脚本读取生效）

| 参数 | 值 | 说明 |
|------|----|------|
| 开票门槛 | 800 | 应付金额 **> 此值** 的普通个人才要求开票核对；**≤ 此值**（含800）不要求开票、直接可付。亮晶姐定：800以下金额小可不开票。 |
| 容差 | 1 | 开票总额与应付金额的差额，绝对值 **≤ 此值** 视为"票款一致"。亮晶姐定：允许几分钱误差（不必完全相等）。觉得 1 元太松可改 0.5 或 0.1。 |
| 实习生关键字 | Intern | 清单「备注」列含此关键字 = 实习生 → 第一轮**豁免**、不标记（在校生劳务不强制开票、不代扣税）。 |
| 外国人识别 | 三信号 | 见第二节。任一命中 → **豁免-外国人**（无法/不要求开中国增值税发票）。 |
| 公司关键字 | 有限公司、有限责任、股份公司、股份有限 | 供应商姓名含任一 → **标黄-待人工**（疑对公/公司收款，勿按个人催票）。 |

> 🔧 改参数：直接改上表的"值"。脚本读不到就用内置默认（门槛800 / 容差1 / Intern / 三信号 / 公司关键字）。

## 二、外国人识别 = 三信号（2026-07 亮晶真实验收补洞）

任一命中即豁免，核对提示写明命中了哪条：

1. **供应商英文名**：供应商姓名去掉括号昵称（如`（明玄）`/`（梵梵）`）后**不含中文且含拉丁字母**。  
   - 覆盖：`AMUSANT Pascal`、`TA THI MINH HUYEN（明玄）`、`Lia fransisca（梵梵）`
2. **开户名英文**：收款「开户名」去掉括号后纯英文。  
   - 覆盖：供应商写中文译名、但银行户名是护照名（如 黎俊灵 → `LE TUAN LINH`；阮青蓝 → `NGUYEN THANH LAM`）
3. **证件为护照号**：清单「身份证号/护照号」**不是**18位中国身份证，且为 5–15 位字母数字（含字母）。  
   - 例：`E04731572`、`C6624835`、`X7636596`

**不误伤**：
- 开户名是中文代收人、供应商是纯英文 → 仍按信号1豁免（代收很常见，如 EMRAH→朱秀梅）。
- 中国人 18 位身份证 + 中文名 + 中文开户 → 不豁免，正常核票。

**仍需人工的灰色**：中文名 + 中文开户 + 填了中方代收人身份证（护照没进系统）→ 三信号都可能 miss，会落到标黄；亮晶看护照后口头说免，可再加备注关键字（暂未做自动）。

## 三、匹配键 = 身份证号为主 + 姓名兜底（亮晶姐 0623 定 · 070 补洞）

- **主匹配**：清单「身份证号/护照号」↔ 发票台账「销售方信息纳税人识别号」（个人通常=身份证号）。
- **为什么主用身份证**：会重名；姓名带特殊字符（如"流畅（阿拉伯语）"）；两张表姓名写法常不一致。
- **姓名兜底（2026-07 加）**：身份证对不上，但台账「销售方信息名称」与清单供应商姓名**精确一致**且有票 → 用姓名汇总金额。  
  - 典型：个体户开票，票上税号是组织税号不是个人身份证（**张佳音** 8593.9）。  
  - 命中时「匹配方式=姓名兜底」，提示写：`身份证未命中，按姓名兜底；票上税号=xxx（疑个体户/税号≠身份证）`。  
  - **已知边界**：极端同名且一人个体户开票时，同名另一人可能误吃到票——国内个人清单极少同名；输出已明示匹配方式，人眼可扫。
- 姓名只用于展示、兜底和人工核对。

## 四、确定性规则（写死在脚本，这里只解释便于核对）

- **开票总额** = 优先按同一身份证号所有发票「合计金额（元）」(含税)**求和**；未命中再按姓名求和。
- **状态判定**顺序：
  1. 备注含实习生关键字 → **豁免-实习生**
  2. 供应商名含公司关键字 **或** 支付渠道含「对公」→ **标黄-待人工**（疑对公，勿进个人催票）
  3. 外国人三信号 → **豁免-外国人**
  4. 应付缺失 → **标黄-待人工**
  5. 应付 ≤ 门槛(800) → **可付**（不要求开票）
  6. 应付 > 门槛：开票总额 ≥ 应付 − 容差 → **可付**（含多开，多开差异留第二轮）；0 < 开票 < 应付 − 容差 → **标黄·缺票**；无匹配发票 → **标黄·未开票**
- **代收提示**：开户名与供应商姓名都是中文且不一致时，不改状态，只在核对提示附加「疑代收」（如谢美群→卜江峰）。
- **差额** = 开票总额 − 应付金额（保留2位）。
- **输出列**：供应商姓名 / 开户名 / 应付金额 / 备注 / 开票总额 / 差额 / 状态 / 匹配方式 / 核对提示。  
  账号/开户行/电话等付款 PII 仍不导出。

## 五、范围与边界（第一阶段）

- 只处理待支付清单的**「国内个人」**（亮晶单独导出；表头常在第2行）；对公供应商本不在内，若混入公司名行会进「待人工」。
- 对公支付的 freelancer（备注写法每次不同）→ **不特殊自动豁免**，没票就标黄、人工再看；公司名行单独待人工。
- 个体户改名/转对公（如朱远辰）→ 台账无个人票时仍标黄，**靠人定走法**，不自动放行。
- 第二阶段（支付后统计有票/无票金额、交思佳做账）留作 v2，本技能不做。
