## 一、可调参数（脚本读取生效）

| 参数 | 值 | 说明 |
|------|----|------|
| 开票门槛 | 800 | 应付金额 **> 此值** 的普通个人才要求开票核对；**≤ 此值**（含800）不要求开票、直接可付。亮晶姐定：800以下金额小可不开票。 |
| 容差 | 1 | 开票总额与应付金额的差额，绝对值 **≤ 此值** 视为"票款一致"。亮晶姐定：允许几分钱误差（不必完全相等）。觉得 1 元太松可改 0.5 或 0.1。 |
| 实习生关键字 | Intern | 清单「备注」列含此关键字 = 实习生 → 第一轮**豁免**、不标记（在校生劳务不强制开票、不代扣税）。 |
| 外国人识别 | 纯英文名 | 「供应商姓名」**不含中文字符** = 外国人 → 豁免（无法开中国增值税发票）。例外（中文名实为外籍、如越南人李双）识别不了 → 会落到"未开票标黄"、人工看护照核实。 |

> 🔧 改参数：直接改上表的"值"。脚本读不到就用内置默认（门槛800 / 容差1 / Intern / 纯英文名）。

## 二、匹配键 = 身份证号（亮晶姐 0623 定）

- 用**身份证号**匹配，不用姓名。清单「身份证号/护照号」 ↔ 发票台账「销售方信息纳税人识别号」（个人=身份证号）。
- 为什么：会重名；姓名带特殊字符（如"流畅（阿拉伯语）"）；两张表姓名写法常不一致。身份证号唯一、最可靠。
- 姓名只用于展示和人工核对兜底。

## 三、确定性规则（写死在脚本，这里只解释便于核对）

- **开票总额** = 发票台账里**同一身份证号**所有发票的「合计金额（元）」(含税)**求和**——解决一人开多张票（VLOOKUP 只能匹一行的老毛病）。
- **状态判定**（普通个人，非实习生/外国人）：
  - 应付 ≤ 门槛(800) → **可付**（不要求开票）。
  - 应付 > 门槛：开票总额 ≥ 应付 − 容差 → **可付**（含开多了的，多开差异留第二轮）；0 < 开票总额 < 应付 − 容差 → **标黄·缺票**（暂停、待补票）；无匹配发票 → **标黄·未开票**（不付、催票）。
  - 身份证能匹配上但开票总额=0（串票/录入0）→ **标黄·未开票**并在「核对提示」标"开票为0待人工核"。
- **差额** = 开票总额 − 应付金额（保留2位）。
- **输出隐藏列**：账号/开户行/开户支行/电话/支付渠道/支付日期 等付款信息列，对核对没用 → 主核对表不展示；身份证号匹配用、输出默认隐藏（PII）。

## 四、范围与边界（第一阶段）

- 只处理待支付清单的**「国内个人」单 sheet**（亮晶单独导出，表头在第2行）；对公供应商不在内。
- 对公支付的 freelancer（郑燕新型，转对公收款）写法每次不同、识别不了 → **不特殊处理**，当普通个人走，没票就标黄、人工再看。
- 第二阶段（支付后统计有票/无票金额、交思佳做账）留作 v2，本技能不做。
